2-SAT encodes "at least one of two literals must be true" constraints via implication: ¬a → b and ¬b → a. Build an implication graph and find SCCs. If variable x and its negation ¬x are in the same SCC, the formula is unsatisfiable. Otherwise, assign truth values based on SCC topological order (x = true iff SCC(x) comes after SCC(¬x) in topological order).
Steps
  1. For each clause (a OR b): add edges (¬a → b) and (¬b → a) to implication graph
  2. Find all SCCs (Kosaraju or Tarjan); label each SCC with topological order index
  3. For each variable i: if scc[i] == scc[¬i]: print "IMPOSSIBLE"
  4. Otherwise: assign true if scc[i] > scc[¬i] (comes later in topological order on condensed DAG)
for (a,b): add ¬a→b and ¬b→a; SCC; val[i]=(scc_order[i]>scc_order[not_i])
Coin Collector
★ EasySCC Condensation + DAG DPO(N + M)
Within each SCC you can collect all coins (you can visit every node in the cycle). So condense each SCC to a single node with coins = sum of all coins in that SCC. The condensation is a DAG (by definition of SCCs). Now find the maximum-weight path in this DAG using DP in topological order, which is straightforward: dp[v] = coins[v] + max(dp[u]) for predecessors u.
Steps
  1. Find all SCCs (Kosaraju/Tarjan); condense into DAG; node weight = sum of coins in SCC
  2. Topological sort the condensed DAG
  3. DP: dp[v] = weight[v] + max(0, max(dp[u] for all edges u→v))
  4. Answer = max(dp[v]) over all nodes in condensed DAG
SCC condensation; DAG longest-weight-path DP in topo order
Mail Delivery
★ EasyEulerian Circuit (Hierholzer's)O(N + M)
An Eulerian circuit (visits every edge exactly once, returns to start) exists in an undirected graph iff the graph is connected AND every vertex has even degree. Both conditions are necessary. If they hold, Hierholzer's algorithm efficiently constructs the circuit: start DFS, and whenever you're at a node with no unvisited edges, add to circuit. The clever trick is using a stack and building the circuit in reverse.
Steps
  1. Check: all nodes with nonzero degree must be connected; all degrees must be even
  2. If not: print "IMPOSSIBLE"
  3. Hierholzer's: DFS with stack; when current node has no unused edges, pop to circuit
  4. Use adjacency list with pointer to track next unused edge efficiently
  5. Print circuit in order (should be N+1... starting and ending at 1)
while stack: v=stack.top; if adj[v] has unused edge: push next; else: circuit.append(stack.pop)
De Bruijn Sequence
★ EasyEulerian Circuit on de Bruijn GraphO(2ⁿ)
A De Bruijn sequence contains every n-bit string exactly once as a substring. Model it as an Eulerian circuit problem: nodes are (n-1)-bit strings, and each n-bit string is a directed edge from its first (n-1) bits to its last (n-1) bits. Every possible edge exists exactly once. An Eulerian circuit on this graph gives the De Bruijn sequence. Each node has equal in-degree and out-degree, so Euler circuit always exists.
Steps
  1. Build a directed graph: 2^(n-1) nodes (all (n-1)-bit strings)
  2. For each n-bit string s: add edge from s[0..n-2] to s[1..n-1]
  3. Each node has in-degree = out-degree = 2 (for binary), so Euler circuit exists
  4. Find Euler circuit starting from node 0; the edges traversed give the De Bruijn sequence
  5. Output: first bit of each edge (or reconstruct the n-bit labels)
nodes=(n-1)-bit strings; edge per n-bit string; Euler circuit = De Bruijn sequence
Teleporters Path
★ EasyEulerian Path (Directed Hierholzer's)O(N + M)
An Eulerian path (visits every edge exactly once, from node 1 to node N) exists in a directed graph iff: exactly one node has out-degree = in-degree+1 (start), exactly one has in-degree = out-degree+1 (end), all others have equal degrees, and the graph is connected. If these conditions hold, Hierholzer's from node 1 gives the path. The conditions enforce 1 is start and N is end.
Steps
  1. Check: node 1 has out = in+1; node N has in = out+1; all others balanced; graph connected
  2. If not: "IMPOSSIBLE"
  3. Hierholzer's DFS from node 1, respecting edge directions
  4. Build circuit by appending when no outgoing edges remain
  5. Reverse and output; should start at 1 and end at N
check degree conditions + connectivity; Hierholzer's directed from node 1
Hamiltonian Flights
★ EasyBitmask DP (Hamiltonian Path)O(2ᴺ · N²)
Hamiltonian path (visiting all nodes exactly once) is NP-hard in general, but with N ≤ 20, bitmask DP is feasible. State: dp[mask][v] = number of paths that visit exactly the nodes in mask and currently end at v. Transition: dp[mask | (1<<u)][u] += dp[mask][v] for each edge v→u. The exponential state space (2^N × N) fits in memory for N=20.
Steps
  1. dp[mask][v] = number of ways to reach node v having visited exactly nodes in mask
  2. Initialize dp[1<<0][0] = 1 (start at node 1, 0-indexed)
  3. For each mask, for each v in mask, for each edge v→u where u not in mask: dp[mask|(1<<u)][u] += dp[mask][v]
  4. Answer = dp[(1<<N)-1][N-1] (visited all, ended at N)
dp[1][0]=1; for mask: for v in mask: for u not in mask: dp[mask|(1&lt;&lt;u)][u]+=dp[mask][v]
Knight's Tour
★ EasyWarnsdorff's HeuristicO(N²) in practice
Finding a Hamiltonian path for a chess knight across an 8×8 board. Brute-force backtracking is too slow without a heuristic. Warnsdorff's rule is a greedy heuristic: always move to the neighbor with the fewest onward moves. This dramatically prunes dead ends by avoiding trapping yourself in low-connectivity corners. It's not guaranteed correct for all starting positions theoretically, but works on 8×8.
Steps
  1. Start from given position, mark visited
  2. At each step, generate all valid unvisited knight moves
  3. Sort candidates by their own number of onward moves (fewest first) — Warnsdorff's rule
  4. Move to the candidate with fewest onward moves; ties can be broken arbitrarily or by secondary sort
  5. Repeat until all 64 squares visited or no moves remain
next_move = min(valid_moves, key=lambda m: count_onward_moves(m))

Distinct Routes
★ EasyMax Flow + Flow DecompositionO(V·E)
Edge-disjoint paths (no two paths share an edge) from 1 to N equals max flow with unit capacities on all edges. After computing max flow, decompose the flow into individual paths: repeatedly DFS from source following edges with positive flow, zeroing flow as you traverse. Each DFS finds one complete path. The number of paths found equals the max flow value.
Steps
  1. Build flow network with unit capacity on each directed edge
  2. Max flow from 1 to N gives number of edge-disjoint paths k
  3. Decompose: for i in 1..k: DFS from node 1, follow edges with remaining flow > 0, set flow=0 on traversed edges
  4. Each DFS ends at N; record the path

Range Queries

Static Range Sum Queries
★ EasyPrefix SumO(N + Q)
Precomputing prefix sums is the textbook approach for repeated range sum queries on a static array. prefix[i] = sum of elements 1..i. Then sum(l,r) = prefix[r] - prefix[l-1]. This converts each O(N) range scan into O(1) arithmetic. The "static" in the problem name signals no updates, so a simple prefix sum suffices without more complex structures.
Steps
  1. Build prefix[0..N] where prefix[i] = prefix[i-1] + a[i]
  2. For each query (l,r): output prefix[r] - prefix[l-1]
prefix[i]=prefix[i-1]+a[i]; query(l,r)=prefix[r]-prefix[l-1]
Static Range Minimum Queries
★ EasySparse TableO(N log N) build, O(1) query
Sparse table achieves O(1) range minimum queries by precomputing minimums for all ranges of length 2^k. The key insight: any range [l,r] can be covered by two overlapping power-of-2 intervals (overlap is fine for min/max since taking min of overlapping regions is idempotent). Query = min(table[l][k], table[r-2^k+1][k]) where 2^k ≤ r-l+1.
Steps
  1. Build sparse_table[i][k] = min of a[i..i+2^k-1]
  2. Fill: sparse_table[i][k] = min(sparse_table[i][k-1], sparse_table[i+2^(k-1)][k-1])
  3. For query (l,r): k = floor(log2(r-l+1)); return min(sparse_table[l][k], sparse_table[r-2^k+1][k])
st[i][k]=min(st[i][k-1], st[i+2^(k-1)][k-1]); query: k=log2(r-l+1); min(st[l][k],st[r-2^k+1][k])
Dynamic Range Sum Queries
★ EasyFenwick Tree (BIT)O((N + Q) log N)
When the array has point updates AND range sum queries, a Fenwick tree (Binary Indexed Tree) is the simplest data structure. It stores partial sums cleverly indexed by lowest-set-bit, allowing both update and prefix-sum query in O(log N). Range sum (l,r) = query(r) - query(l-1). The BIT is memory-efficient and cache-friendly compared to segment trees.
Steps
  1. BIT of size N, initially built from the array
  2. Point update at i: propagate change up via i += i & (-i)
  3. Prefix sum query up to i: accumulate down via i -= i & (-i)
  4. Range query (l,r) = prefix(r) - prefix(l-1)
update(i,d): while i&lt;=N: bit[i]+=d; i+=i&(-i); query(i): s=0; while i>0: s+=bit[i]; i-=i&(-i)
Dynamic Range Minimum Queries
★ EasySegment TreeO((N + Q) log N)
Point updates + range min queries require a segment tree (BIT doesn't support range min efficiently because min isn't invertible like sum). Each segment tree node stores the minimum of its range. Point update walks down to a leaf and propagates minimums back up. Range min query combines O(log N) nodes that together cover the query range. This is the canonical segment tree problem.
Steps
  1. Build segment tree: leaves = a[i]; internal nodes = min of children
  2. Point update at i: update leaf, then update ancestors up the tree
  3. Range query (l,r): recursively split into left/right children, return min of overlapping parts
tree[node]=min(tree[left],tree[right]); update propagates up; query merges min down
Range Xor Queries
★ EasyPrefix XORO(N + Q)
XOR has the same prefix-sum property as addition because XOR is its own inverse (a XOR a = 0). Prefix XOR array lets you compute range XOR in O(1): xor(l,r) = prefix[r] XOR prefix[l-1]. This is because prefix[r] XOR prefix[l-1] cancels out all elements before l, leaving exactly the XOR of elements l..r.
Steps
  1. Build prefix_xor[0..N]: prefix_xor[i] = prefix_xor[i-1] XOR a[i]
  2. For each query (l,r): output prefix_xor[r] XOR prefix_xor[l-1]
px[i]=px[i-1]^a[i]; query(l,r)=px[r]^px[l-1]
Range Update Queries
★ EasyBIT on Difference ArrayO((N + Q) log N)
Range add + point query is the "inverse" of the point update + range sum problem. The key insight: use a difference array where adding v to range [l,r] becomes two point updates: diff[l] += v and diff[r+1] -= v. The actual value at position i is the prefix sum of the difference array. A BIT on the difference array supports both operations efficiently.
Steps
  1. BIT representing the difference array, initially all zeros
  2. Range update (l,r,v): BIT.update(l, +v) and BIT.update(r+1, -v)
  3. Point query at i: output a[i] + BIT.prefix_sum(i) (original value + accumulated updates)
range_add(l,r,v): bit.update(l,+v); bit.update(r+1,-v); point_query(i): a[i]+bit.prefix(i)
Forest Queries
★ Easy2D Prefix SumO(N² + Q)
Counting trees in a rectangle of a 2D grid is a 2D version of prefix sums. Precompute prefix[r][c] = number of trees in the rectangle from (1,1) to (r,c). Then any rectangle query uses inclusion-exclusion: prefix[r2][c2] - prefix[r1-1][c2] - prefix[r2][c1-1] + prefix[r1-1][c1-1]. The 2D inclusion-exclusion prevents double-subtraction of the top-left corner.
Steps
  1. Build 2D prefix: prefix[r][c] = prefix[r-1][c] + prefix[r][c-1] - prefix[r-1][c-1] + (grid[r][c]=='*')
  2. For each query (r1,c1,r2,c2): answer = prefix[r2][c2] - prefix[r1-1][c2] - prefix[r2][c1-1] + prefix[r1-1][c1-1]
p[r][c]=p[r-1][c]+p[r][c-1]-p[r-1][c-1]+(grid=='*'); query=p[r2][c2]-p[r1-1][c2]-p[r2][c1-1]+p[r1-1][c1-1]

★ EasySegment Tree DescentO((N + Q) log N)
For each group of size k, you need to find the leftmost hotel with at least k free rooms — a classic segment tree "descent" operation. The segment tree stores max free rooms in each range. To find the leftmost position with value ≥ k: if left child's max ≥ k, recurse left; otherwise recurse right. This navigates down in O(log N) per query.
Steps
  1. Segment tree where each node = max of its range (leaf = rooms[i])
  2. For each group k: descend tree starting from root to find leftmost index with value ≥ k
  3. At each node: if left_child.max ≥ k, go left; else go right
  4. If root.max < k: no hotel available (problem likely guarantees this won't happen)

Subordinates
★ EasyDFS Subtree SizeO(N)
In a rooted tree, the number of subordinates of node v is exactly the size of its subtree minus 1 (excluding v itself). A simple DFS computes subtree sizes: size[v] = 1 + sum of size[child] for all children. The root has N-1 subordinates, leaves have 0. This is the most fundamental tree DP.
Steps
  1. Root the tree at node 1, DFS to compute subtree sizes
  2. size[v] = 1 + sum(size[child]) for all children of v
  3. Output size[v] - 1 for each node v

★ EasyDouble BFS/DFSO(N)

This works because: start from any node, the farthest node must be one endpoint of a diameter (provable by contradiction). Then from that endpoint, the farthest node is the other diameter endpoint. The distance between these two is the diameter.

  2. BFS/DFS from u; find farthest node v and distance

This is because the farthest node from any v must be one of the two diameter endpoints (provable: if it were some other node w, extending through the diameter would create a longer path). So find both diameter endpoints (via the two-BFS trick), compute distances from each, and take the max per node.

  3. BFS from v: get dist_v[i] for all i

Tree Distances II
★ EasyRerooting DPO(N)
Compute the sum of distances from every node to all other nodes. First DFS computes subtree sizes and sum of distances from root (downward). The second pass uses "rerooting": when moving root from v to child c, nodes in c's subtree get 1 closer (size[c] nodes) and all others get 1 farther (N - size[c] nodes). So: dist_sum[c] = dist_sum[v] - size[c] + (N - size[c]).

Company Queries I
★ EasyBinary LiftingO(N log N) build, O(log N) query
Finding the k-th ancestor of a node in a tree naively requires k parent-pointer hops (O(k)). Binary lifting precomputes up[v][j] = 2^j-th ancestor of v in O(N log N). To find the k-th ancestor, decompose k in binary: if bit j is set, jump up[v][j] and clear that bit. This is "fast exponentiation" applied to tree ancestor queries.
Steps
  1. up[v][0] = parent[v]; up[v][j] = up[up[v][j-1]][j-1] for j=1..log(N)
  2. For query (v, k): iterate j from high to low; if (k >> j) & 1: v = up[v][j]
  3. If v becomes root (no such ancestor): output -1 if needed
up[v][j]=up[up[v][j-1]][j-1]; query: for j=LOG..0: if k>>j&1: v=up[v][j]
Company Queries II
★ EasyLCA via Binary LiftingO(N log N) build, O(log N) query
LCA (Lowest Common Ancestor) of two nodes u and v is their deepest common ancestor. With binary lifting: first equalize depths by lifting the deeper node, then lift both simultaneously until they meet. The LCA is the point where they first converge. This requires the same up[][] table as Company Queries I, making it a natural extension.
Steps
  1. Compute depth[] and binary lifting table up[][]
  2. LCA(u,v): if depth[u] < depth[v]: swap; lift u by depth difference using binary lifting
  3. If u == v: return u (one was ancestor of other)
  4. Lift both u,v simultaneously as high as possible without meeting; return up[u][0]

Path Queries
★ EasyEuler Tour + BIT (Range Update / Point Query)O((N + Q) log N)
Path query (sum of node values from root to v) uses an interesting Euler tour property: when updating node v's value, it affects all paths from root to any node in v's subtree. We have a partial bracket prefix and need to count valid completions. First check the prefix never goes negative.   2. Let b = final balance, r = 2n - prefix_length; if r-b is odd or negative, answer is 0

  4. Answer = C(r, open) - C(r, open-1) mod MOD

Counting Necklaces
★★★ HardBurnside's LemmaO(n log n)
Counting distinct necklaces under rotation is the canonical Burnside's lemma application: |distinct objects| = (1/|G|) Σ_{g∈G} |Fix(g)|. For a necklace of n beads with m colors, the rotation group has n elements. A rotation by d positions fixes exactly m^gcd(n,d) colorings — each cycle of beads forced to the same color, and gcd(n,d) is the cycle count.

  2. Sum all m^gcd(n,d) values mod MOD
  3. Multiply by modular inverse of n

Counting Grids
★★★ HardBurnside's Lemma (Rotation Group)O(log n)
Counting distinct n×n binary grids under 90°/180°/270°/identity rotation uses Burnside with 4 group elements. Each rotation has different cycle structure on grid cells: identity has n² cycles, 180° has ⌈n²/2⌉ cycles, and 90°/270° have ⌈n²/4⌉ cycles each. Fixed colorings for each = 2^(cycle count).   2. 180° rotation: cycles = ceil(n*n/2), fixed = 2^cycles
  3. 90° and 270° rotations: cycles = ceil(n*n/4), fixed = 2^cycles each

  g[n] = mex(reachable);
}
