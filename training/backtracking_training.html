Backtracking & Recursion

Subsets • Permutations • Combinations • Pruning • Constraint Satisfaction • DFS Patterns

When to Use Backtracking

Recognition Signals

"Find all..." / "Generate all..." / "List every possible..."
"Print all permutations/combinations/subsets"
"Is there a valid assignment?" (Sudoku, N-Queens)
"Partition into groups satisfying..."

Backtracking vs Other Approaches

Problem TypeApproachWhy
Find ALL solutionsBacktrackingMust enumerate
Count solutionsDP (if overlapping subproblems)Avoid re-enumeration
Find ONE solutionBacktracking + early returnStop at first valid
Optimal valueDP or GreedyDon't need all solutions
Constraint satisfactionBacktracking + pruningSearch space is finite

Complexity

Subsets: O(2^n) — each element in or out
Permutations: O(n!) — n choices, then n-1, then n-2...
Combinations C(n,k): O(C(n,k)) — choose k from n
With pruning: much less in practice, but worst case unchanged

Core Backtracking Template

The Universal Pattern

void backtrack(State state, List<Result> results) {
    // Base case: found a complete solution
    if (isComplete(state)) {
        results.add(copy(state));
        return;
    }

    // Iterate over all choices at current decision point
    for (Choice choice : getChoices(state)) {
        // Prune: skip invalid choices early
        if (!isValid(state, choice)) continue;

        // Make choice
        state.apply(choice);

        // Recurse
        backtrack(state, results);

        // Undo choice (backtrack)
        state.undo(choice);
    }
}

Key Decisions When Designing

DecisionOptionsImpact
What is "state"?Path so far, remaining items, current positionDetermines memory usage
What are "choices"?Next element to add, position to fill, value to tryDetermines branching factor
When is it "complete"?Path length = n, all positions filled, target reachedDetermines tree depth
How to prune?Skip duplicates, bound checking, constraint propagationCritical for performance

Two Flavors: Choose/Not-Choose vs Iterate-Choices

// Flavor 1: For each element, choose to include or exclude
// Good for: subsets, partition problems
void backtrack(int index) {
    if (index == n) { collect(); return; }
    // Include nums[index]
    path.add(nums[index]);
    backtrack(index + 1);
    path.removeLast();
    // Exclude nums[index]
    backtrack(index + 1);
}

// Flavor 2: At each step, try all remaining candidates
// Good for: permutations, combinations with constraints
void backtrack(int start) {
    collect();  // every partial state is valid (for subsets)
    for (int i = start; i < n; i++) {
        path.add(nums[i]);
        backtrack(i + 1);  // i+1 for combinations, i for reuse
        path.removeLast();
    }
}

Subsets (Power Set)

Basic Subsets (LC 78)

// All subsets of distinct elements
List<List<Integer>> result = new ArrayList<>();

void backtrack(int[] nums, int start, List<Integer> path) {
    result.add(new ArrayList<>(path));  // every state is a valid subset
    for (int i = start; i < nums.length; i++) {
        path.add(nums[i]);
        backtrack(nums, i + 1, path);
        path.remove(path.size() - 1);
    }
}

Subsets with Duplicates (LC 90)

// Sort first, then skip consecutive duplicates at same level
Arrays.sort(nums);

void backtrack(int[] nums, int start, List<Integer> path) {
    result.add(new ArrayList<>(path));
    for (int i = start; i < nums.length; i++) {
        if (i > start && nums[i] == nums[i-1]) continue;  // skip dup!
        path.add(nums[i]);
        backtrack(nums, i + 1, path);
        path.remove(path.size() - 1);
    }
}

Deduplication Rule

Sort + "skip if same as previous at same decision level":
if (i > start && nums[i] == nums[i-1]) continue;
This ensures we only pick the FIRST occurrence of each value at each recursion level. Works for subsets, combinations, and permutations.

Permutations

Basic Permutations (LC 46)

// All permutations of distinct elements
void backtrack(int[] nums, boolean[] used, List<Integer> path) {
    if (path.size() == nums.length) {
        result.add(new ArrayList<>(path));
        return;
    }
    for (int i = 0; i < nums.length; i++) {
        if (used[i]) continue;
        used[i] = true;
        path.add(nums[i]);
        backtrack(nums, used, path);
        path.remove(path.size() - 1);
        used[i] = false;
    }
}

Permutations with Duplicates (LC 47)

// Sort + skip rule: only use a duplicate if the previous copy was used
Arrays.sort(nums);

void backtrack(int[] nums, boolean[] used, List<Integer> path) {
    if (path.size() == nums.length) { result.add(new ArrayList<>(path)); return; }
    for (int i = 0; i < nums.length; i++) {
        if (used[i]) continue;
        if (i > 0 && nums[i] == nums[i-1] && !used[i-1]) continue; // key!
        used[i] = true;
        path.add(nums[i]);
        backtrack(nums, used, path);
        path.remove(path.size() - 1);
        used[i] = false;
    }
}

Swap-based Permutation (in-place, O(1) extra)

// Swap element at index with all positions >= index
void backtrack(int[] nums, int start) {
    if (start == nums.length) { result.add(toList(nums)); return; }
    for (int i = start; i < nums.length; i++) {
        swap(nums, start, i);
        backtrack(nums, start + 1);
        swap(nums, start, i);  // undo
    }
}

Combinations

Choose k from n (LC 77)

// C(n, k): choose k elements from [1..n]
void backtrack(int start, int n, int k, List<Integer> path) {
    if (path.size() == k) { result.add(new ArrayList<>(path)); return; }
    for (int i = start; i <= n - (k - path.size()) + 1; i++) { // pruning!
        path.add(i);
        backtrack(i + 1, n, k, path);
        path.remove(path.size() - 1);
    }
}

Combination Sum (LC 39 — unlimited use)

// Each number can be used unlimited times, find all combos summing to target
void backtrack(int[] candidates, int start, int remain, List<Integer> path) {
    if (remain == 0) { result.add(new ArrayList<>(path)); return; }
    for (int i = start; i < candidates.length; i++) {
        if (candidates[i] > remain) break;  // sort first for this pruning
        path.add(candidates[i]);
        backtrack(candidates, i, remain - candidates[i], path); // i, not i+1
        path.remove(path.size() - 1);
    }
}

Combination Sum II (LC 40 — each used once, with duplicates)

// Sort + skip duplicates at same level
void backtrack(int[] candidates, int start, int remain, List<Integer> path) {
    if (remain == 0) { result.add(new ArrayList<>(path)); return; }
    for (int i = start; i < candidates.length; i++) {
        if (candidates[i] > remain) break;
        if (i > start && candidates[i] == candidates[i-1]) continue; // dedup
        path.add(candidates[i]);
        backtrack(candidates, i + 1, remain - candidates[i], path);
        path.remove(path.size() - 1);
    }
}

String & Array Partitioning

Palindrome Partitioning (LC 131)

// Partition string so every part is a palindrome
void backtrack(String s, int start, List<String> path) {
    if (start == s.length()) { result.add(new ArrayList<>(path)); return; }
    for (int end = start + 1; end <= s.length(); end++) {
        String sub = s.substring(start, end);
        if (isPalindrome(sub)) {
            path.add(sub);
            backtrack(s, end, path);
            path.remove(path.size() - 1);
        }
    }
}

Restore IP Addresses (LC 93)

// Partition string into exactly 4 valid IP segments (0-255)
void backtrack(String s, int start, int parts, List<String> path) {
    if (parts == 4 && start == s.length()) { result.add(String.join(".", path)); return; }
    if (parts == 4 || start == s.length()) return;
    for (int len = 1; len <= 3 && start + len <= s.length(); len++) {
        String seg = s.substring(start, start + len);
        if (seg.length() > 1 && seg.charAt(0) == '0') break; // no leading zero
        if (Integer.parseInt(seg) > 255) break;
        path.add(seg);
        backtrack(s, start + len, parts + 1, path);
        path.remove(path.size() - 1);
    }
}

Constraint Satisfaction (N-Queens, Sudoku)

N-Queens (LC 51)

// Place N queens on N×N board, no two attack each other
boolean[] cols, diag1, diag2;  // track attacked columns & diagonals

void backtrack(int row) {
    if (row == n) { result.add(buildBoard()); return; }
    for (int col = 0; col < n; col++) {
        if (cols[col] || diag1[row-col+n] || diag2[row+col]) continue;
        queens[row] = col;
        cols[col] = diag1[row-col+n] = diag2[row+col] = true;
        backtrack(row + 1);
        cols[col] = diag1[row-col+n] = diag2[row+col] = false;
    }
}

Sudoku Solver (LC 37)

// Fill empty cells, checking row/col/box constraints
boolean solve(char[][] board) {
    for (int i = 0; i < 9; i++) {
        for (int j = 0; j < 9; j++) {
            if (board[i][j] != '.') continue;
            for (char c = '1'; c <= '9'; c++) {
                if (isValid(board, i, j, c)) {
                    board[i][j] = c;
                    if (solve(board)) return true;
                    board[i][j] = '.';  // backtrack
                }
            }
            return false;  // no valid digit works here
        }
    }
    return true;  // all cells filled
}

Pruning Techniques

Pruning is what makes backtracking practical. Without it, you search the entire tree. With it, you cut branches early.

Common Pruning Strategies

StrategyWhenHow
Sort + Skip DuplicatesInput has duplicatesif (i > start && a[i] == a[i-1]) continue
Early TerminationRemaining can't reach goalif (remain < 0) return
Upper BoundBest possible from here < current bestBranch and bound
Constraint PropagationChoices constrain futureN-Queens: track cols/diags
Sort DescendingLarger items first = fewer branchesFail faster on sum problems
Symmetry BreakingSymmetric solutionsFix first choice, halve search

Example: Combination Sum with Sorting Prune

// Without sorting: must check all candidates even if too large
// With sorting: once candidates[i] > remain, break (all further are bigger)
Arrays.sort(candidates);
for (int i = start; i < candidates.length; i++) {
    if (candidates[i] > remain) break;  // prune entire subtree!
    ...
}

Example: K-Sum Pruning

// Need k numbers summing to target from sorted array
// Prune: if smallest k remaining > target, impossible
// Prune: if largest k remaining < target, impossible
if (nums[start] * k > remain) return;  // min possible too big
if (nums[n-1] * k < remain) return;  // max possible too small

Grid-Based Backtracking

Word Search (LC 79)

boolean dfs(char[][] board, String word, int i, int j, int k) {
    if (k == word.length()) return true;
    if (i < 0 || i >= m || j < 0 || j >= n) return false;
    if (board[i][j] != word.charAt(k)) return false;

    char tmp = board[i][j];
    board[i][j] = '#';  // mark visited

    boolean found = dfs(board, word, i+1, j, k+1)
                  || dfs(board, word, i-1, j, k+1)
                  || dfs(board, word, i, j+1, k+1)
                  || dfs(board, word, i, j-1, k+1);

    board[i][j] = tmp;  // unmark (backtrack)
    return found;
}

Robot Room Cleaner (LC 489)

Pattern: DFS with relative direction. At each cell: try all 4 directions. If can move, recurse. Backtrack by turning 180° and moving back. Key: track visited using (relative row, relative col).

Interview Tips

How to Communicate

1. "This asks for all valid solutions, so I'll use backtracking."
2. "My state is [path built so far]. My choices at each step are [remaining candidates]."
3. "I can prune by [sorting + skip duplicates / early termination when sum exceeds target]."
4. "Time complexity is O(2^n) for subsets / O(n!) for permutations, which is optimal since we need all solutions."

Common Mistakes

Forgetting to copy: result.add(new ArrayList<>(path)) not result.add(path)
Wrong loop start: Combinations use start, permutations use 0
Missing dedup: Sort + skip same value at same level
Not restoring state: Every add must have matching remove
Infinite recursion: Ensure progress toward base case
