When to Use Which Traversal

Binary tree problems account for ~15-20% of Google interviews. Choosing the right traversal order is half the battle.

Traversal Selection Guide

Problem TypeTraversalWhy
BST operations (validate, search, kth)InorderInorder of BST = sorted order
Serialize / clone / reconstructPreorderRoot-first makes reconstruction natural
Delete nodes / free memoryPostorderProcess children before parent
Shortest path / level-by-levelLevel-order (BFS)Natural level separation
Path sum / diameter / heightPostorder (bottom-up)Need subtree info before deciding at node
Top-down propagation (path from root)Preorder (top-down)Pass info from parent to children

Complexity Reference

OperationTimeSpace
Any traversal (recursive)O(n)O(h) stack
Any traversal (iterative)O(n)O(h) stack
Level-order (BFS)O(n)O(w) queue, w = max width
Morris traversalO(n)O(1)
Build tree from traversalsO(n)O(n) hashmap
LCA (binary lifting)O(log n) per queryO(n log n) preprocessing

Key insight: Most binary tree problems boil down to one question: "What info do I need from subtrees (postorder) vs. what info do I pass down (preorder)?"

class TreeNode {

Traversals

Master all four traversals in both recursive and iterative form. This is non-negotiable for Google interviews.

Google interviewers may ask you to convert between recursive and iterative on the spot. Practice writing both forms without hesitation.

Inorder (Left, Root, Right)

Recursive

void inorder(TreeNode node, List<Integer> res) {
    if (node == null) return;
    inorder(node.left, res);
    res.add(node.val);
    inorder(node.right, res);
}

Iterative (Stack)

List<Integer> inorderIterative(TreeNode root) {
    List<Integer> res = new ArrayList<>();
    Deque<TreeNode> stack = new ArrayDeque<>();
    TreeNode cur = root;
    while (cur != null || !stack.isEmpty()) {
        while (cur != null) {
            stack.push(cur);
            cur = cur.left;
        }
        cur = stack.pop();
        res.add(cur.val);
        cur = cur.right;
    }
    return res;
}

Morris Inorder (O(1) space)

Rarely asked directly, but good to know. It modifies the tree temporarily by threading right pointers, then restores it.

List<Integer> morrisInorder(TreeNode root) {
    List<Integer> res = new ArrayList<>();
    TreeNode cur = root;
    while (cur != null) {
        if (cur.left == null) {
            res.add(cur.val);
            cur = cur.right;
        } else {
            // Find inorder predecessor
            TreeNode pred = cur.left;
            while (pred.right != null && pred.right != cur) {
                pred = pred.right;
            }
            if (pred.right == null) {
                // Thread: make predecessor point back to cur
                pred.right = cur;
                cur = cur.left;
            } else {
                // Unthread: restore tree, visit cur
                pred.right = null;
                res.add(cur.val);
                cur = cur.right;
            }
        }
    }
    return res;
}

Preorder (Root, Left, Right)

Recursive

void preorder(TreeNode node, List<Integer> res) {
    if (node == null) return;
    res.add(node.val);
    preorder(node.left, res);
    preorder(node.right, res);
}

Iterative (Stack)

List<Integer> preorderIterative(TreeNode root) {
    List<Integer> res = new ArrayList<>();
    if (root == null) return res;
    Deque<TreeNode> stack = new ArrayDeque<>();
    stack.push(root);
    while (!stack.isEmpty()) {
        TreeNode node = stack.pop();
        res.add(node.val);
        // Push right first so left is processed first
        if (node.right != null) stack.push(node.right);
        if (node.left != null) stack.push(node.left);
    }
    return res;
}

Postorder (Left, Right, Root)

Recursive

void postorder(TreeNode node, List<Integer> res) {
    if (node == null) return;
    postorder(node.left, res);
    postorder(node.right, res);
    res.add(node.val);
}

Iterative (Two Stacks / Reverse Preorder)

// Trick: modified preorder (Root, Right, Left) then reverse
List<Integer> postorderIterative(TreeNode root) {
    LinkedList<Integer> res = new LinkedList<>();
    if (root == null) return res;
    Deque<TreeNode> stack = new ArrayDeque<>();
    stack.push(root);
    while (!stack.isEmpty()) {
        TreeNode node = stack.pop();
        res.addFirst(node.val); // add to front
        if (node.left != null) stack.push(node.left);
        if (node.right != null) stack.push(node.right);
    }
    return res;
}

Iterative (Single Stack, True Postorder)

List<Integer> postorderSingleStack(TreeNode root) {
    List<Integer> res = new ArrayList<>();
    Deque<TreeNode> stack = new ArrayDeque<>();
    TreeNode cur = root, lastVisited = null;
    while (cur != null || !stack.isEmpty()) {
        while (cur != null) {
            stack.push(cur);
            cur = cur.left;
        }
        TreeNode peek = stack.peek();
        if (peek.right != null && peek.right != lastVisited) {
            cur = peek.right;
        } else {
            res.add(peek.val);
            lastVisited = stack.pop();
        }
    }
    return res;
}

Level-Order (BFS)

Recursive (DFS with depth tracking)

void levelOrderDFS(TreeNode node, int depth, List<List<Integer>> res) {
    if (node == null) return;
    if (depth == res.size()) res.add(new ArrayList<>());
    res.get(depth).add(node.val);
    levelOrderDFS(node.left, depth + 1, res);
    levelOrderDFS(node.right, depth + 1, res);
}

Iterative (Queue)

List<List<Integer>> levelOrder(TreeNode root) {
    List<List<Integer>> res = new ArrayList<>();
    if (root == null) return res;
    Queue<TreeNode> q = new LinkedList<>();
    q.offer(root);
    while (!q.isEmpty()) {
        int size = q.size();
        List<Integer> level = new ArrayList<>();
        for (int i = 0; i < size; i++) {
            TreeNode node = q.poll();
            level.add(node.val);
            if (node.left != null) q.offer(node.left);
            if (node.right != null) q.offer(node.right);
        }
        res.add(level);
    }
    return res;
}

Pattern: int size = q.size() before the inner loop is the key trick for level separation in BFS.

Construction & Serialization

Building trees from traversal arrays and serializing/deserializing are classic Google problems. The key insight: you need TWO traversals (one must be inorder) to uniquely define a tree, OR you can serialize with null markers.

Build from Preorder + Inorder (LC 105)

int preIdx = 0;
Map<Integer, Integer> inMap = new HashMap<>();

TreeNode buildTree(int[] preorder, int[] inorder) {
    for (int i = 0; i < inorder.length; i++)
        inMap.put(inorder[i], i);
    return build(preorder, 0, inorder.length - 1);
}

TreeNode build(int[] preorder, int inLeft, int inRight) {
    if (inLeft > inRight) return null;
    int rootVal = preorder[preIdx++];
    TreeNode root = new TreeNode(rootVal);
    int idx = inMap.get(rootVal);
    root.left = build(preorder, inLeft, idx - 1);
    root.right = build(preorder, idx + 1, inRight);
    return root;
}

Why it works: Preorder gives us the root. Inorder tells us which elements are in the left vs. right subtree (everything left of root in inorder = left subtree).

Build from Inorder + Postorder (LC 106)

int postIdx;
Map<Integer, Integer> inMap = new HashMap<>();

TreeNode buildTree(int[] inorder, int[] postorder) {
    postIdx = postorder.length - 1;
    for (int i = 0; i < inorder.length; i++)
        inMap.put(inorder[i], i);
    return build(postorder, 0, inorder.length - 1);
}

TreeNode build(int[] postorder, int inLeft, int inRight) {
    if (inLeft > inRight) return null;
    int rootVal = postorder[postIdx--];
    TreeNode root = new TreeNode(rootVal);
    int idx = inMap.get(rootVal);
    // Build RIGHT first (postorder processes right before left in reverse)
    root.right = build(postorder, idx + 1, inRight);
    root.left = build(postorder, inLeft, idx - 1);
    return root;
}

Serialize / Deserialize (LC 297)

// Preorder with null markers
public String serialize(TreeNode root) {
    StringBuilder sb = new StringBuilder();
    serDFS(root, sb);
    return sb.toString();
}

void serDFS(TreeNode node, StringBuilder sb) {
    if (node == null) { sb.append("#,"); return; }
    sb.append(node.val).append(",");
    serDFS(node.left, sb);
    serDFS(node.right, sb);
}

public TreeNode deserialize(String data) {
    Queue<String> q = new LinkedList<>(Arrays.asList(data.split(",")));
    return desDFS(q);
}

TreeNode desDFS(Queue<String> q) {
    String val = q.poll();
    if (val.equals("#")) return null;
    TreeNode node = new TreeNode(Integer.parseInt(val));
    node.left = desDFS(q);
    node.right = desDFS(q);
    return node;
}

BST from Preorder (LC 1008)

int idx = 0;

TreeNode bstFromPreorder(int[] preorder) {
    return build(preorder, Integer.MAX_VALUE);
}

TreeNode build(int[] preorder, int upperBound) {
    if (idx == preorder.length || preorder[idx] > upperBound)
        return null;
    TreeNode root = new TreeNode(preorder[idx++]);
    root.left = build(preorder, root.val);
    root.right = build(preorder, upperBound);
    return root;
}

BST property eliminates the need for inorder! The preorder alone suffices because we know left children are smaller, right children are larger.

Path Problems

Path problems use a common pattern: compute the answer at each node using subtree information (postorder), and track the global best with an instance variable.

Binary Tree Maximum Path Sum (LC 124)

int maxSum = Integer.MIN_VALUE;

int maxPathSum(TreeNode root) {
    dfs(root);
    return maxSum;
}

// Returns max "one-arm" path sum starting at node going down
int dfs(TreeNode node) {
    if (node == null) return 0;
    int left = Math.max(0, dfs(node.left));   // clamp negative to 0
    int right = Math.max(0, dfs(node.right));
    // Update global: path through this node as "turning point"
    maxSum = Math.max(maxSum, left + right + node.val);
    // Return one-arm to parent
    return Math.max(left, right) + node.val;
}

Common mistake: Forgetting to clamp negative subtree sums to 0. A negative subtree should be ignored, not included in the path.

Diameter of Binary Tree (LC 543)

int diameter = 0;

int diameterOfBinaryTree(TreeNode root) {
    height(root);
    return diameter;
}

int height(TreeNode node) {
    if (node == null) return 0;
    int left = height(node.left);
    int right = height(node.right);
    diameter = Math.max(diameter, left + right); // edges, not nodes
    return Math.max(left, right) + 1;
}

Path Sum III (LC 437) - Prefix Sum on Tree

int count = 0;

int pathSum(TreeNode root, int targetSum) {
    Map<Long, Integer> prefixMap = new HashMap<>();
    prefixMap.put(0L, 1);
    dfs(root, 0L, targetSum, prefixMap);
    return count;
}

void dfs(TreeNode node, long curSum, int target, Map<Long, Integer> map) {
    if (node == null) return;
    curSum += node.val;
    count += map.getOrDefault(curSum - target, 0);
    map.merge(curSum, 1, Integer::sum);
    dfs(node.left, curSum, target, map);
    dfs(node.right, curSum, target, map);
    map.merge(curSum, -1, Integer::sum); // backtrack
}

Pattern recognition: "Path Sum III" is just the "Subarray Sum Equals K" technique (prefix sum + hashmap) applied to tree paths from root to node.

Longest Univalue Path (LC 687)

int longest = 0;

int longestUnivaluePath(TreeNode root) {
    dfs(root);
    return longest;
}

int dfs(TreeNode node) {
    if (node == null) return 0;
    int left = dfs(node.left);
    int right = dfs(node.right);
    int arrowLeft = 0, arrowRight = 0;
    if (node.left != null && node.left.val == node.val)
        arrowLeft = left + 1;
    if (node.right != null && node.right.val == node.val)
        arrowRight = right + 1;
    longest = Math.max(longest, arrowLeft + arrowRight);
    return Math.max(arrowLeft, arrowRight);
}

Lowest Common Ancestor (LCA)

LCA is one of the most frequently asked binary tree problems at Google. Know all three variants cold.

LCA of Binary Tree (LC 236) - Recursive

TreeNode lowestCommonAncestor(TreeNode root, TreeNode p, TreeNode q) {
    if (root == null || root == p || root == q) return root;
    TreeNode left = lowestCommonAncestor(root.left, p, q);
    TreeNode right = lowestCommonAncestor(root.right, p, q);
    if (left != null && right != null) return root;
    return left != null ? left : right;
}

How this works: If both left and right subtrees return non-null, current node is the LCA. If only one side returns non-null, the LCA is in that subtree (or is that returned node itself).

LCA of Binary Tree - Iterative with Parent Pointers

TreeNode lcaIterative(TreeNode root, TreeNode p, TreeNode q) {
    Map<TreeNode, TreeNode> parent = new HashMap<>();
    parent.put(root, null);
    Deque<TreeNode> stack = new ArrayDeque<>();
    stack.push(root);
    // Step 1: Build parent map until both p and q are found
    while (!parent.containsKey(p) || !parent.containsKey(q)) {
        TreeNode node = stack.pop();
        if (node.left != null) {
            parent.put(node.left, node);
            stack.push(node.left);
        }
        if (node.right != null) {
            parent.put(node.right, node);
            stack.push(node.right);
        }
    }
    // Step 2: Collect ancestors of p
    Set<TreeNode> ancestors = new HashSet<>();
    while (p != null) {
        ancestors.add(p);
        p = parent.get(p);
    }
    // Step 3: First ancestor of q that's also ancestor of p
    while (!ancestors.contains(q)) {
        q = parent.get(q);
    }
    return q;
}

LCA of BST (LC 235)

// Exploit BST property: if both p,q < node, go left; both > node, go right
TreeNode lowestCommonAncestor(TreeNode root, TreeNode p, TreeNode q) {
    TreeNode cur = root;
    while (cur != null) {
        if (p.val < cur.val && q.val < cur.val)
            cur = cur.left;
        else if (p.val > cur.val && q.val > cur.val)
            cur = cur.right;
        else
            return cur; // split point = LCA
    }
    return null;
}

LCA with Parent Pointer (LC 1650)

// Same as "Intersection of Two Linked Lists" (LC 160)
Node lowestCommonAncestor(Node p, Node q) {
    Node a = p, b = q;
    while (a != b) {
        a = (a != null) ? a.parent : q;
        b = (b != null) ? b.parent : p;
    }
    return a;
}

Elegant trick: When two pointers reach their respective "end" (root's parent = null), redirect to the other's starting point. They meet at LCA because they travel equal total distance.

Views & Boundaries

These problems test your ability to define traversal order precisely and handle edge cases with coordinates or depths.

Right Side View (LC 199)

List<Integer> rightSideView(TreeNode root) {
    List<Integer> res = new ArrayList<>();
    dfs(root, 0, res);
    return res;
}

void dfs(TreeNode node, int depth, List<Integer> res) {
    if (node == null) return;
    if (depth == res.size()) res.add(node.val);
    dfs(node.right, depth + 1, res); // right first!
    dfs(node.left, depth + 1, res);
}

Key trick: Visit right child first. The first node encountered at each depth is the rightmost one. For left view, visit left first instead.

Boundary of Binary Tree (LC 545)

List<Integer> boundaryOfBinaryTree(TreeNode root) {
    List<Integer> res = new ArrayList<>();
    if (root == null) return res;
    res.add(root.val);
    leftBoundary(root.left, res);
    leaves(root.left, res);
    leaves(root.right, res);
    rightBoundary(root.right, res);
    return res;
}

void leftBoundary(TreeNode node, List<Integer> res) {
    while (node != null) {
        if (!isLeaf(node)) res.add(node.val);
        node = (node.left != null) ? node.left : node.right;
    }
}

void rightBoundary(TreeNode node, List<Integer> res) {
    Deque<Integer> stack = new ArrayDeque<>();
    while (node != null) {
        if (!isLeaf(node)) stack.push(node.val);
        node = (node.right != null) ? node.right : node.left;
    }
    while (!stack.isEmpty()) res.add(stack.pop());
}

void leaves(TreeNode node, List<Integer> res) {
    if (node == null) return;
    if (isLeaf(node)) { res.add(node.val); return; }
    leaves(node.left, res);
    leaves(node.right, res);
}

boolean isLeaf(TreeNode node) {
    return node.left == null && node.right == null;
}

Vertical Order Traversal (LC 987)

List<List<Integer>> verticalTraversal(TreeNode root) {
    // TreeMap<col, TreeMap<row, PriorityQueue>>
    TreeMap<Integer, TreeMap<Integer, PriorityQueue<Integer>>> map = new TreeMap<>();
    dfs(root, 0, 0, map);
    List<List<Integer>> res = new ArrayList<>();
    for (var colMap : map.values()) {
        List<Integer> col = new ArrayList<>();
        for (var pq : colMap.values()) {
            while (!pq.isEmpty()) col.add(pq.poll());
        }
        res.add(col);
    }
    return res;
}

void dfs(TreeNode node, int row, int col,
         TreeMap<Integer, TreeMap<Integer, PriorityQueue<Integer>>> map) {
    if (node == null) return;
    map.computeIfAbsent(col, k -> new TreeMap<>())
       .computeIfAbsent(row, k -> new PriorityQueue<>())
       .offer(node.val);
    dfs(node.left, row + 1, col - 1, map);
    dfs(node.right, row + 1, col + 1, map);
}

LC 314 vs LC 987: They look similar but have different tie-breaking rules. LC 314 uses insertion order (BFS) while LC 987 sorts by value when row and column are equal.

Tree Modification

Problems that require restructuring the tree. The key is understanding how pointer manipulation works in a recursive context.

Invert Binary Tree (LC 226)

TreeNode invertTree(TreeNode root) {
    if (root == null) return null;
    TreeNode left = invertTree(root.left);
    TreeNode right = invertTree(root.right);
    root.left = right;
    root.right = left;
    return root;
}

Flatten Binary Tree to Linked List (LC 114)

// In-place, preorder, right pointers form the linked list
void flatten(TreeNode root) {
    TreeNode cur = root;
    while (cur != null) {
        if (cur.left != null) {
            // Find rightmost node in left subtree
            TreeNode pred = cur.left;
            while (pred.right != null) pred = pred.right;
            // Connect it to cur.right
            pred.right = cur.right;
            // Move left subtree to right
            cur.right = cur.left;
            cur.left = null;
        }
        cur = cur.right;
    }
}

Alternative approach (reverse postorder): Process right, left, root. Maintain a prev pointer. Set node.right = prev; node.left = null; prev = node.

Populating Next Right Pointers (LC 116 - Perfect BT)

// O(1) space using already-established next pointers
Node connect(Node root) {
    Node levelStart = root;
    while (levelStart != null && levelStart.left != null) {
        Node cur = levelStart;
        while (cur != null) {
            cur.left.next = cur.right;
            if (cur.next != null)
                cur.right.next = cur.next.left;
            cur = cur.next;
        }
        levelStart = levelStart.left;
    }
    return root;
}

Populating Next Right Pointers II (LC 117 - Any BT)

Node connect(Node root) {
    Node cur = root;
    while (cur != null) {
        Node dummy = new Node(0); // dummy for next level
        Node tail = dummy;
        while (cur != null) {
            if (cur.left != null) { tail.next = cur.left; tail = tail.next; }
            if (cur.right != null) { tail.next = cur.right; tail = tail.next; }
            cur = cur.next;
        }
        cur = dummy.next; // move to next level
    }
    return root;
}

BST Operations

BST problems exploit the ordering invariant: left.val < node.val < right.val. Inorder traversal of a BST yields sorted order.

Validate BST (LC 98)

boolean isValidBST(TreeNode root) {
    return validate(root, Long.MIN_VALUE, Long.MAX_VALUE);
}

boolean validate(TreeNode node, long min, long max) {
    if (node == null) return true;
    if (node.val <= min || node.val >= max) return false;
    return validate(node.left, min, node.val)
        && validate(node.right, node.val, max);
}

Pitfall: Using int bounds fails when node values are Integer.MIN_VALUE or MAX_VALUE. Always use long for bounds.

BST Iterator (LC 173)

class BSTIterator {
    Deque<TreeNode> stack = new ArrayDeque<>();

    BSTIterator(TreeNode root) {
        pushLeft(root);
    }

    int next() {
        TreeNode node = stack.pop();
        pushLeft(node.right);
        return node.val;
    }

    boolean hasNext() {
        return !stack.isEmpty();
    }

    void pushLeft(TreeNode node) {
        while (node != null) {
            stack.push(node);
            node = node.left;
        }
    }
}

O(h) space, O(1) amortized next(). This is the controlled version of iterative inorder traversal. Each node is pushed and popped exactly once across all next() calls.

Kth Smallest Element in BST (LC 230)

int kthSmallest(TreeNode root, int k) {
    Deque<TreeNode> stack = new ArrayDeque<>();
    TreeNode cur = root;
    while (cur != null || !stack.isEmpty()) {
        while (cur != null) {
            stack.push(cur);
            cur = cur.left;
        }
        cur = stack.pop();
        if (--k == 0) return cur.val;
        cur = cur.right;
    }
    return -1; // shouldn't reach here
}

Delete Node in BST (LC 450)

TreeNode deleteNode(TreeNode root, int key) {
    if (root == null) return null;
    if (key < root.val) {
        root.left = deleteNode(root.left, key);
    } else if (key > root.val) {
        root.right = deleteNode(root.right, key);
    } else {
        // Found the node to delete
        if (root.left == null) return root.right;
        if (root.right == null) return root.left;
        // Two children: replace with inorder successor
        TreeNode succ = root.right;
        while (succ.left != null) succ = succ.left;
        root.val = succ.val;
        root.right = deleteNode(root.right, succ.val);
    }
    return root;
}

Recover BST (LC 99)

// Two nodes are swapped. Find them via inorder (detect inversions).
TreeNode first, second, prev;

void recoverTree(TreeNode root) {
    inorder(root);
    // Swap values of the two misplaced nodes
    int tmp = first.val;
    first.val = second.val;
    second.val = tmp;
}

void inorder(TreeNode node) {
    if (node == null) return;
    inorder(node.left);
    if (prev != null && prev.val > node.val) {
        if (first == null) first = prev;
        second = node;
    }
    prev = node;
    inorder(node.right);
}

Two cases: If swapped nodes are adjacent in inorder, there's one inversion. If non-adjacent, there are two inversions. The code handles both: first is set on the first inversion, second is updated on every inversion.

Interview Tips

Recursive vs Iterative Conversion Strategy

When asked to convert recursive to iterative:

  1. Identify what the call stack stores (current node, state of which children have been processed)
  2. Replace implicit stack with explicit stack (Deque/ArrayDeque in Java)
  3. Simulate the "return" via a loop condition (while stack not empty)
  4. For postorder: track "last visited" or use the reverse-preorder trick

Don't overthink it. For preorder, just push right then left. For inorder, use the "go left as far as possible" pattern. Postorder is the tricky one - use the reverse trick unless they insist on true iterative.

Common Patterns

Pattern 1: Return value = "info from subtree"

The recursive function returns something that helps the parent make decisions.

// Example: height, max path going down, validity info
int height(TreeNode node) {
    if (node == null) return 0;
    return 1 + Math.max(height(node.left), height(node.right));
}
// Used in: diameter, balanced check, max path sum

Pattern 2: Global variable for "best so far"

When the answer isn't at the root but could be at any node.

int best = 0; // or Integer.MIN_VALUE

int solve(TreeNode node) {
    // ... compute local answer using subtree info
    best = Math.max(best, localAnswer);
    // ... return something useful to parent
}
// Used in: max path sum (124), diameter (543), longest univalue (687)

Pattern 3: Pass info down (top-down)

Parent passes constraints or accumulated state to children via parameters.

void dfs(TreeNode node, int depth, long min, long max) {
    // Use depth for level tracking, min/max for BST validation
}
// Used in: validate BST (98), path sum (112), right view (199)

Edge Cases Checklist

  • Empty tree (root == null) - always handle first
  • Single node - no children, height = 0 or 1 depending on definition
  • Skewed tree (linked list) - O(n) height, stack overflow risk for recursion
  • Negative values - affects path sum problems (can't prune early)
  • Duplicate values - affects BST operations (strict vs. non-strict inequality)
  • Integer overflow - use long for bounds in BST validation

Time Management in Interviews

For binary tree problems at Google:
- 2 min: clarify input/output, edge cases, confirm approach
- 3 min: explain your approach (traversal type, what you return/pass)
- 10 min: code the solution
- 3 min: trace through an example
- 2 min: discuss complexity, potential follow-ups

Common Follow-Up Questions

  • "Can you do it iteratively?" (traversal conversion)
  • "What if the tree is very deep?" (iterative to avoid stack overflow)
  • "What if nodes don't have unique values?" (affects construction, LCA)
  • "Can you do it in O(1) space?" (Morris traversal, parent pointers)
  • "What if this is a BST?" (exploit ordering for O(h) solutions)
