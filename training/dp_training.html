How to Recognize a DP Problem

Two Core Properties

  1. Optimal Substructure — The optimal solution can be constructed from optimal solutions of its subproblems.
  2. Overlapping Subproblems — The same subproblem is solved multiple times during recursion.

Quick Signal Words

High probability DP when you see: "minimum", "maximum", "longest", "shortest", "number of ways", "is it possible", "count all", "optimal cost under constraint"

Decision Tree

Q1: Does the problem ask to enumerate ALL permutations/combinations? └— Yes → Backtracking (not DP) └— No → Q2: Does it have Optimal Substructure? (answer = f(sub-answers)) └— No → Greedy / BFS / Other └— Yes → Q3: Do subproblems overlap? └— No → Divide & Conquer └— Yes → Dynamic Programming

DP vs Greedy vs Backtracking

The 5-Step DP Framework

1. Define State — What does dp[i] represent? Say it in one sentence.
2. Transition — How to compute dp[i] from smaller subproblems?
3. Base Case — What are the trivial sub-answers?
4. Computation Order — Bottom-up or Top-down? Ensure dependencies are met.
5. Answer Location — Is the final answer dp[n]? dp[n][m]? max(dp[...])?

1D Dynamic Programming

State has one dimension: dp[i] typically represents "the optimal answer considering the first i elements".

Generic Template

int[] dp = new int[n + 1];
dp[0] = base_case;

for (int i = 1; i <= n; i++) {
    for (int choice : choices) {
        dp[i] = Math.max(dp[i], dp[prev] + cost);
    }
}
return dp[n];

Subtype 1: Fibonacci-like (sum of fixed previous states)

// LC 70 Climbing Stairs
// dp[i] = number of ways to reach step i
dp[i] = dp[i-1] + dp[i-2];

Subtype 2: Take-or-Skip (House Robber pattern)

// LC 198 House Robber
// dp[i] = max profit considering first i houses
dp[i] = Math.max(dp[i-1], dp[i-2] + nums[i]);
// skip house i  vs  rob house i

Subtype 3: LIS-like (enumerate all j < i)

// LC 300 Longest Increasing Subsequence
// dp[i] = length of LIS ending at nums[i]
for (int i = 0; i < n; i++) {
    dp[i] = 1;
    for (int j = 0; j < i; j++) {
        if (nums[j] < nums[i])
            dp[i] = Math.max(dp[i], dp[j] + 1);
    }
}

Subtype 4: Jump-forward (transition from i to i+k)

// LC 1326 Minimum Number of Taps
// dp[i] = minimum operations to cover position i
for (int i = 0; i <= n; i++) {
    for (reachable position j from i) {
        dp[j] = Math.min(dp[j], dp[i] + 1);
    }
}

2D Dynamic Programming

State has two dimensions. Common scenarios: grid traversal, comparing two sequences, or adding a constraint dimension.

Type A: Grid Path

// LC 62 Unique Paths / LC 64 Minimum Path Sum
// dp[i][j] = min cost (or path count) from (0,0) to (i,j)
for (int i = 0; i < m; i++) {
    for (int j = 0; j < n; j++) {
        if (i == 0 && j == 0) dp[i][j] = grid[0][0];
        else dp[i][j] = Math.min(
            i > 0 ? dp[i-1][j] : INF,
            j > 0 ? dp[i][j-1] : INF
        ) + grid[i][j];
    }
}

Type B: Two-Sequence Comparison

// LC 1143 Longest Common Subsequence
// dp[i][j] = LCS length of s[0..i-1] and t[0..j-1]
for (int i = 1; i <= m; i++) {
    for (int j = 1; j <= n; j++) {
        if (s.charAt(i-1) == t.charAt(j-1))
            dp[i][j] = dp[i-1][j-1] + 1;
        else
            dp[i][j] = Math.max(dp[i-1][j], dp[i][j-1]);
    }
}

Type C: 1D + Extra Constraint Dimension

// LC 123 Best Time to Buy and Sell Stock III
// dp[i][k] = max profit at day i with k transactions done
dp[i][k][0] = Math.max(dp[i-1][k][0], dp[i-1][k][1] + prices[i]);
dp[i][k][1] = Math.max(dp[i-1][k][1], dp[i-1][k-1][0] - prices[i]);

Space Optimization: Rolling Array

Key idea: If dp[i] only depends on dp[i-1], compress to one row. Reverse iteration order to avoid overwriting needed values.

// Before: dp[i][j] depends on dp[i-1][j] and dp[i-1][j-1]
// After: single array + reverse traversal
for (int i = 1; i <= m; i++)
    for (int j = n; j >= 1; j--)  // reverse!
        dp[j] = dp[j] + dp[j-1];

3D Dynamic Programming

When 2D is insufficient to capture state. Common in: dual-path, multi-operation, or extra-constraint problems.

Type A: Dual-Path (Cherry Pickup)

// LC 741 Cherry Pickup
// Two people walk from (0,0) to (n-1,n-1) simultaneously
// dp[r1][c1][r2] -- c2 = r1+c1-r2 (same step count)
for (int step = 0; step < 2*n-2; step++) {
    for (int r1 ...) {
        for (int r2 ...) {
            // 4 combos: (down,down),(down,right),(right,down),(right,right)
            dp[r1][c1][r2] = max of 4 prev states + cherries;
        }
    }
}

Type B: K Transactions (Stock Series)

// LC 188 Best Time to Buy and Sell Stock IV
// dp[i][k][hold] = day i, k transactions done, holding or not
int[][][] dp = new int[n][K+1][2];
for (int i = 1; i < n; i++) {
    for (int k = 1; k <= K; k++) {
        dp[i][k][0] = Math.max(dp[i-1][k][0], dp[i-1][k][1] + prices[i]);
        dp[i][k][1] = Math.max(dp[i-1][k][1], dp[i-1][k-1][0] - prices[i]);

Knapsack DP

Given a capacity constraint, select items to maximize value. The most classic DP family.

0/1 Knapsack (each item used at most once)

// dp[i][w] = max value using first i items with capacity w
// Transition: take or skip item i
dp[i][w] = Math.max(dp[i-1][w], dp[i-1][w - weight[i]] + value[i]);

// Space-optimized (1D, reverse order)
for (int i = 0; i < n; i++)
    for (int w = W; w >= weight[i]; w--)
        dp[w] = Math.max(dp[w], dp[w - weight[i]] + value[i]);

Unbounded Knapsack (each item unlimited)

// Difference: forward iteration (allows repeated selection)
for (int i = 0; i < n; i++)
    for (int w = weight[i]; w <= W; w++)  // forward!
        dp[w] = Math.max(dp[w], dp[w - weight[i]] + value[i]);

Knapsack Variants

Count ways: dp[w] += dp[w - weight[i]] (not max)
Feasibility: dp[w] = dp[w] || dp[w - weight[i]]
Exactly fill W: init dp[0]=0, rest=-INF
At most W: init all to 0

Common Transformation

Target Sum (LC 494): P - N = target, P + N = sum, so P = (sum + target) / 2. Convert to 0/1 knapsack counting ways with capacity P.

Combinations vs Permutations (Loop Order!)

This is the #1 coin change trap. The two-loop order determines the answer semantics:

// LC 518: How many COMBINATIONS sum to target?
// {1,2} and {2,1} are the SAME → count once
// Coins OUTER, amount INNER:
for (int coin : coins)            // consider each coin in sequence
    for (int j = coin; j <= target; j++)
        dp[j] += dp[j - coin];

// LC 377: How many PERMUTATIONS (ordered) sum to target?
// {1,2} and {2,1} are DIFFERENT → count both
// Amount OUTER, coins INNER:
for (int j = 1; j <= target; j++)  // for each amount, try ALL coins
    for (int coin : coins)
        if (j >= coin) dp[j] += dp[j - coin];

How to never get this wrong (intuition, not memorization):
• Coins outer = "I decide coin1's contribution, then coin2's, then coin3's..." — order of coins is fixed → no duplicate orderings → combinations
• Amount outer = "At each amount, I can pick ANY coin" — so reaching 3 via [1,2] and [2,1] are separate decisions → permutations

Interview phrasing: "If I loop coins outside, I process each denomination in a fixed sequence, so I'll never count choosing coin A after coin B AND coin B after coin A — that gives combinations. If I loop amount outside, at every amount I'm free to pick any coin again, which creates ordered sequences."

String DP

DP on one or two strings. State dp[i][j] usually represents a relationship between s[0..i-1] and t[0..j-1].

Template A: Two-String Comparison (LCS / Edit Distance)

// dp[i][j] = answer for s[0..i-1] vs t[0..j-1]
for (int i = 1; i <= m; i++) {
    for (int j = 1; j <= n; j++) {
        if (s.charAt(i-1) == t.charAt(j-1))
            dp[i][j] = dp[i-1][j-1] + 1;
        else
            dp[i][j] = Math.max(dp[i-1][j], dp[i][j-1]);
    }
}

Template B: Single-String Palindrome (Interval-style)

// LC 516 Longest Palindromic Subsequence
// dp[i][j] = longest palindromic subseq in s[i..j]
for (int len = 2; len <= n; len++) {
    for (int i = 0; i + len - 1 < n; i++) {
        int j = i + len - 1;
        if (s.charAt(i) == s.charAt(j))
            dp[i][j] = dp[i+1][j-1] + 2;
        else
            dp[i][j] = Math.max(dp[i+1][j], dp[i][j-1]);
    }
}

Template C: Regex / Wildcard Matching

// LC 10 Regular Expression Matching
// dp[i][j] = s[0..i-1] matches p[0..j-1]
if (p.charAt(j-1) == '*') {
    dp[i][j] = dp[i][j-2];  // zero occurrence
    if (match(s, p, i, j-1))
        dp[i][j] |= dp[i-1][j];  // one+ occurrence
} else {
    dp[i][j] = match(s, p, i, j) && dp[i-1][j-1];
}

Interval DP

State dp[l][r] = optimal answer for the interval [l, r]. Enumerate split point k in [l, r-1].

Core Template

// enumerate length -> left endpoint -> split point
for (int len = 2; len <= n; len++) {
    for (int l = 0; l + len - 1 < n; l++) {
        int r = l + len - 1;
        dp[l][r] = INF;
        for (int k = l; k < r; k++) {
            dp[l][r] = Math.min(dp[l][r],
                dp[l][k] + dp[k+1][r] + cost(l, k, r));
        }
    }
}

Recognition Signals

Merge adjacent elements → Stone merge, Matrix chain multiplication
Pop/remove then neighbors merge → Burst Balloons
Operate on both ends → Stone Game
Palindrome (interval shrinks) → Palindrome Partitioning

Tree DP

DP on tree structures. Typically uses post-order DFS — children report results upward.

Core Template

// dp[node][0] = optimal NOT selecting node
// dp[node][1] = optimal selecting node
void dfs(TreeNode node) {
    if (node == null) return;
    dfs(node.left);
    dfs(node.right);

    // Don't select: children can be either
    dp[node][0] = Math.max(dp[left][0], dp[left][1])
               + Math.max(dp[right][0], dp[right][1]);
    // Select: children must NOT be selected
    dp[node][1] = node.val + dp[left][0] + dp[right][0];
}

Graph DP

DP on graphs. Prerequisite: graph must be a DAG, or use topological sort / bitmask for cycles.

Template A: Longest/Shortest Path on DAG

// Topological sort + DP
// dp[node] = longest path to reach node
for (int node : topologicalOrder) {
    for (int next : adj[node]) {
        dp[next] = Math.max(dp[next], dp[node] + weight[node][next]);
    }
}

Template B: All-Pairs Shortest Path (Floyd-Warshall)

// dp[k][i][j] = shortest i->j using first k nodes as intermediaries
for (int k = 0; k < n; k++)
    for (int i = 0; i < n; i++)
        for (int j = 0; j < n; j++)
            dp[i][j] = Math.min(dp[i][j], dp[i][k] + dp[k][j]);

Template C: Stateful Graph DP

// LC 787 Cheapest Flights Within K Stops
// dp[k][node] = min cost to reach node with at most k stops
for (int step = 1; step <= K+1; step++) {
    for (int[] edge : flights) {
        int u = edge[0], v = edge[1], w = edge[2];
        dp[step][v] = Math.min(dp[step][v], dp[step-1][u] + w);
    }
}

Digit DP

For counting problems: "How many numbers in [L, R] satisfy property X?" Build number digit by digit.

Core Idea

count(R) - count(L-1): Convert range query to "count of numbers ≤ X".
Build digit by digit from MSB to LSB, tracking tight (whether still bounded by upper limit).

Generic Template

// dp[pos][tight][state] -- memoized recursion
int[][][] memo;
String num;  // upper bound as string

int solve(int pos, boolean tight, int state) {
    if (pos == num.length()) return isValid(state) ? 1 : 0;
    if (memo[pos][tight?1:0][state] != -1)
        return memo[pos][tight?1:0][state];

    int limit = tight ? (num.charAt(pos) - '0') : 9;
    int res = 0;

    for (int d = 0; d <= limit; d++) {
        res += solve(pos + 1,
                     tight && (d == limit),
                     newState(state, d));
    }

    memo[pos][tight?1:0][state] = res;
    return res;
}

Bitmask DP (State Compression)

Use a binary bitmask to represent "which elements have been selected". Works when n ≤ 20.

Core Template

// dp[mask] = optimal answer when selected set = mask
// n elements, 2^n total states
int[] dp = new int[1 << n];
Arrays.fill(dp, INF);
dp[0] = 0;

for (int mask = 0; mask < (1 << n); mask++) {
    int bits = Integer.bitCount(mask);
    for (int j = 0; j < n; j++) {
        if ((mask & (1 << j)) == 0) {  // j not selected
            int newMask = mask | (1 << j);
            dp[newMask] = Math.min(dp[newMask],
                dp[mask] + cost[bits][j]);
        }
    }
}

TSP (Traveling Salesman)

// dp[mask][i] = shortest path visiting cities in mask, at city i
int[][] dp = new int[1 << n][n];
// Transition: from i, go to unvisited j
dp[mask | (1 << j)][j] = Math.min(..., dp[mask][i] + dist[i][j]);

When to Use

n ≤ 20 (2^20 ~ 1M states, acceptable)
Track "which selected/visited"
Permutation assignment: n people → n tasks
Set cover: select fewest subsets to cover universe

Sorting + DP

Sort first to eliminate one dimension of comparison, then apply DP on the remaining problem.

Why Sort First?

Eliminate bidirectional comparison — after sorting, only one direction matters.
Greedy + DP fusion — sorting provides locally optimal processing order.
Reduce to LIS — sort one dimension, the other becomes LIS.

Classic: Russian Doll Envelopes

// LC 354: envelopes (w, h), find max nesting depth
// Sort: w ascending; same w -> h descending
// Then LIS on h (w already non-decreasing)
Arrays.sort(envelopes, (a, b) ->
    a[0] != b[0] ? a[0] - b[0] : b[1] - a[1]);

// h descending ensures same-w picks at most one
int[] h = extractHeights(envelopes);
return lengthOfLIS(h);  // O(n log n)

Job Scheduling Pattern

// LC 1235 Maximum Profit in Job Scheduling
// Sort by end time, dp[i] = max profit first i jobs
Arrays.sort(jobs, (a, b) -> a.end - b.end);
dp[i] = Math.max(
    dp[i-1],  // skip job i
    dp[binarySearch(lastNonOverlap)] + profit[i]  // take
);

State Machine DP

Model the problem as states with transitions. Each state represents a "mode" (holding stock, in cooldown, etc). Track the best value for each state at each step.

Core Idea

Define states (e.g., hold, not-hold, cooldown) and transitions between them. At each step, compute each state's optimal value from previous step's states. The answer is the best among final states.

Template: Stock with Cooldown (LC 309)

// States: hold (have stock), sold (just sold), rest (cooldown done)
int hold = -prices[0], sold = 0, rest = 0;

for (int i = 1; i < n; i++) {
    int prevHold = hold, prevSold = sold, prevRest = rest;
    hold = Math.max(prevHold, prevRest - prices[i]);  // keep or buy
    sold = prevHold + prices[i];                       // sell
    rest = Math.max(prevRest, prevSold);               // wait
}
return Math.max(sold, rest);

Template: Stock with Transaction Fee (LC 714)

// Two states: hold, cash (not holding)
int hold = -prices[0], cash = 0;

for (int i = 1; i < n; i++) {
    hold = Math.max(hold, cash - prices[i]);
    cash = Math.max(cash, hold + prices[i] - fee);
}
return cash;

Template: At Most K Transactions (LC 188)

// dp[k][0] = max profit with k transactions, not holding
// dp[k][1] = max profit with k transactions, holding
int[][] dp = new int[K + 1][2];
for (int k = 0; k <= K; k++) dp[k][1] = Integer.MIN_VALUE;

for (int price : prices) {
    for (int k = K; k >= 1; k--) {
        dp[k][0] = Math.max(dp[k][0], dp[k][1] + price);    // sell
        dp[k][1] = Math.max(dp[k][1], dp[k-1][0] - price);  // buy
    }
}
return dp[K][0];

General State Machine Pattern

// LC 1220 Count Vowels Permutation
// States: which vowel is the last character
// Transitions: defined by rules (e.g., 'a' can follow 'e','i','u')
long[] dp = {1,1,1,1,1}; // a, e, i, o, u

for (int step = 1; step < n; step++) {
    long[] next = new long[5];
    next[0] = (dp[1] + dp[2] + dp[4]) % MOD;  // a <- e,i,u
    next[1] = (dp[0] + dp[2]) % MOD;           // e <- a,i
    next[2] = (dp[1] + dp[3]) % MOD;           // i <- e,o
    next[3] = (dp[2]) % MOD;                   // o <- i
    next[4] = (dp[2] + dp[3]) % MOD;           // u <- i,o
    dp = next;
}

When to Recognize State Machine DP

Multiple "modes" or "states" at each position (holding/not, cooldown, last action)
Transitions depend on current state + decision (buy, sell, rest)
Finite, small set of states (2-5 typically)
Examples: Stock series, House Robber (take/skip), Paint House (colors), Vowel Permutation

BIT (Binary Indexed Tree) + DP

When DP transitions require fast prefix max/sum queries, use BIT or Segment Tree to accelerate from O(n²) to O(n log n).

Core Application: LIS in O(n log n)

// Naive LIS O(n^2): dp[i] = max(dp[j]+1) for j<i, nums[j]<nums[i]
// BIT: use value domain as index, query max dp in [0, nums[i]-1]

int[] sorted = compress(nums);  // map values to [1, n]
BIT bit = new BIT(n);

for (int i = 0; i < n; i++) {
    int val = sorted[i];
    int best = bit.query(val - 1);  // max LIS for values < nums[i]
    bit.update(val, best + 1);     // update LIS at nums[i]
}

BIT Structure

class BIT {
    int[] tree;
    int n;

    BIT(int n) { this.n = n; tree = new int[n + 1]; }

    void update(int i, int val) {
        for (; i <= n; i += i & (-i))
            tree[i] = Math.max(tree[i], val);
    }

    int query(int i) {
        int res = 0;
        for (; i > 0; i -= i & (-i))
            res = Math.max(res, tree[i]);
        return res;
    }
}

When Needed

Pattern: dp[i] = max/min(dp[j]) + ... for j in some range
Brute force: O(n) per transition → O(n²) total
With BIT: O(log n) per transition → O(n log n) total

Monotone Stack / Deque Optimization

When DP transitions involve sliding-window max/min, use a monotone deque for O(1) amortized access.

Monotone Deque Template

// dp[i] = min(dp[j] + cost(j,i)) for j in [i-k, i-1]
Deque<Integer> deque = new ArrayDeque<>();

for (int i = 0; i < n; i++) {
    // Remove elements outside window
    while (!deque.isEmpty() && deque.peekFirst() < i - k)
        deque.pollFirst();

    // Transition: use deque front (optimal j)
    dp[i] = dp[deque.peekFirst()] + cost;

    // Maintain monotonicity
    while (!deque.isEmpty() && dp[deque.peekLast()] >= dp[i])
        deque.pollLast();
    deque.offerLast(i);
}

Monotone Stack: Largest Rectangle

// LC 84 Largest Rectangle in Histogram
Deque<Integer> stack = new ArrayDeque<>();
for (int i = 0; i <= n; i++) {
    int h = (i == n) ? 0 : heights[i];
    while (!stack.isEmpty() && heights[stack.peek()] > h) {
        int height = heights[stack.pop()];
        int width = stack.isEmpty() ? i : i - stack.peek() - 1;
        maxArea = Math.max(maxArea, height * width);
    }
    stack.push(i);
}

Probability DP

State represents a probability or expected value. Transitions multiply/divide by probability of each event. Use double[] or track fractions.

Template: Knight Probability (LC 688)

// dp[r][c] = probability of being at (r,c) after k moves
double[][] dp = new double[n][n];
dp[row][col] = 1.0;

for (int step = 0; step < k; step++) {
    double[][] next = new double[n][n];
    for (int i = 0; i < n; i++)
        for (int j = 0; j < n; j++)
            if (dp[i][j] > 0)
                for (int[] dir : dirs) {
                    int x = i + dir[0], y = j + dir[1];
                    if (x >= 0 && x < n && y >= 0 && y < n)
                        next[x][y] += dp[i][j] / 8.0;
                }
    dp = next;
}
// Sum all dp[i][j] = total probability of staying on board

Template: New 21 Game (LC 837)

// dp[i] = probability of reaching score exactly i
// Draw uniformly from [1, maxPts], stop when score >= k
double[] dp = new double[n + 1];
dp[0] = 1.0;
double windowSum = 1.0;

for (int i = 1; i <= n; i++) {
    dp[i] = windowSum / maxPts;
    if (i < k) windowSum += dp[i];   // still drawing
    if (i >= maxPts) windowSum -= dp[i - maxPts];
}
// Answer = sum(dp[k..n])

Game Theory DP (Minimax)

Two players take turns. Each plays optimally. Determine winner or optimal score difference.

Template: Stone Game (LC 877 / 1140)

// dp[i][j] = max score difference (current player - opponent) for piles[i..j]
for (int len = 1; len <= n; len++) {
    for (int i = 0; i + len - 1 < n; i++) {
        int j = i + len - 1;
        if (len == 1) dp[i][j] = piles[i];
        else dp[i][j] = Math.max(
            piles[i] - dp[i+1][j],   // take left
            piles[j] - dp[i][j-1]    // take right
        );
    }
}
// First player wins if dp[0][n-1] > 0

Template: Minimax with Memoization

// LC 464 Can I Win / LC 1728 Cat and Mouse II
Map<Integer, Boolean> memo = new HashMap<>();

boolean canWin(int state) {
    if (memo.containsKey(state)) return memo.get(state);

    for (each possible move) {
        int nextState = applyMove(state, move);
        if (isWin(nextState) || !canWin(nextState)) {
            memo.put(state, true);
            return true;  // current player can force a win
        }
    }
    memo.put(state, false);
    return false;
}

Key Insight: Score Difference Trick

Instead of tracking both players' scores separately:
dp[state] = (current player's score) - (opponent's score from remaining game).
When current player takes value V: dp = V - dp[next_state].
This works because opponent's best = our worst, and vice versa.

Recursive ↔ Iterative Conversion (Google loves this!)

Google interviewers WILL ask you to convert. If you write bottom-up, they may say "can you do it recursively?" or vice versa. Be ready to do both instantly.

// BOTTOM-UP (iterative):
int[] dp = new int[n + 1];
for (int i = 1; i <= n; i++)
    for (int coin : coins)
        if (i >= coin) dp[i] = Math.min(dp[i], dp[i - coin] + 1);

// TOP-DOWN (recursive) — exact same logic, different structure:
int[] memo = new int[n + 1];
Arrays.fill(memo, -1);

int solve(int amount) {
    if (amount == 0) return 0;
    if (amount < 0) return Integer.MAX_VALUE;
    if (memo[amount] != -1) return memo[amount];
    int res = Integer.MAX_VALUE;
    for (int coin : coins)
        res = Math.min(res, solve(amount - coin) + 1);
    return memo[amount] = res;
}

2D → 1D Space Optimization (Common Follow-up)

"Can you reduce space from O(mn) to O(n)?"
If dp[i][j] only depends on dp[i-1][...], keep only one row. Check dependency direction to decide iteration order.

// 2D version: O(m*n) space
int[][] dp = new int[m+1][n+1];
for (int i = 1; i <= m; i++)
    for (int j = 1; j <= n; j++)
        dp[i][j] = dp[i-1][j-1] + 1;  // depends on prev row

// 1D version: O(n) space — iterate j in REVERSE if depends on dp[i-1][j-1]
int[] dp = new int[n+1];
for (int i = 1; i <= m; i++)
    for (int j = n; j >= 1; j--)  // REVERSE to avoid overwriting!
        dp[j] = dp[j-1] + 1;

// Rules:
// - Depends on dp[i-1][j] and dp[i-1][j-1] → reverse j
// - Depends on dp[i][j-1] (same row, left) → forward j is fine
// - Depends on both → need a "prev" variable to save dp[j-1] before overwrite

Advanced: Knuth's Optimization

Reduces interval DP from O(n³) to O(n²) when optimal split is monotone. Applies to LC 1547 (Cut Stick). Won't be asked in interviews — interviewers likely don't know it either — but useful for competitive programming.

Common Pitfalls

Missing base case: dp[0] = ? Often 0 or 1, think carefully.
Wrong iteration order: 0/1 knapsack reverse; unbounded forward.
Ambiguous state: "first i items" vs "ending at i" — huge difference!
Integer overflow: Counting problems need mod 10^9+7.
Boundary: i-1 < 0? j-weight < 0? Always check.
