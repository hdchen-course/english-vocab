BFS & DFS

BFS Template (Level-order, shortest path in unweighted)

Queue<Integer> queue = new LinkedList<>();
boolean[] visited = new boolean[n];
queue.offer(start);
visited[start] = true;
int level = 0;

while (!queue.isEmpty()) {
    int size = queue.size();
    for (int i = 0; i < size; i++) {
        int node = queue.poll();
        for (int neighbor : graph.get(node)) {
            if (!visited[neighbor]) {
                visited[neighbor] = true;
                queue.offer(neighbor);
            }
        }
    }
    level++;
}

DFS Template (Iterative)

Deque<Integer> stack = new ArrayDeque<>();
boolean[] visited = new boolean[n];
stack.push(start);

while (!stack.isEmpty()) {
    int node = stack.pop();
    if (visited[node]) continue;
    visited[node] = true;
    for (int neighbor : graph.get(node))
        if (!visited[neighbor])
            stack.push(neighbor);
}

Topological Sort

Linear ordering of vertices in a DAG such that for every edge u→v, u comes before v. Only exists if graph has no cycles.

When to Use

Signal words: "prerequisite", "dependency", "ordering", "schedule", "before/after"
Key insight: If the problem involves ordering with constraints (A must come before B), it's topological sort.

Method 1: Kahn's Algorithm (BFS-based) — PREFERRED

// O(V + E) — also detects cycles (if result.size() < n, cycle exists)
int[] topoSort(int n, List<List<Integer>> graph) {
    int[] indegree = new int[n];
    for (int u = 0; u < n; u++)
        for (int v : graph.get(u))
            indegree[v]++;

    Queue<Integer> queue = new LinkedList<>();
    for (int i = 0; i < n; i++)
        if (indegree[i] == 0) queue.offer(i);

    int[] order = new int[n];
    int idx = 0;

    while (!queue.isEmpty()) {
        int u = queue.poll();
        order[idx++] = u;
        for (int v : graph.get(u)) {
            indegree[v]--;
            if (indegree[v] == 0) queue.offer(v);
        }
    }

    return idx == n ? order : new int[0];  // empty = cycle
}

Method 2: DFS-based (Post-order reverse)

boolean[] visited = new boolean[n];
boolean[] inStack = new boolean[n];  // for cycle detection
Deque<Integer> stack = new ArrayDeque<>();
boolean hasCycle = false;

void dfs(int u) {
    visited[u] = true;
    inStack[u] = true;
    for (int v : graph.get(u)) {
        if (inStack[v]) { hasCycle = true; return; }
        if (!visited[v]) dfs(v);
    }
    inStack[u] = false;
    stack.push(u);  // post-order
}

// Call dfs for all unvisited nodes, then stack = topo order

// Process nodes in topological order, relax edges
int[] dist = new int[n];
Arrays.fill(dist, Integer.MIN_VALUE);

Variant: All Topological Orders (Backtracking)

// Used when problem asks "all valid orderings"
void allTopoSort(List<Integer> current) {
    if (current.size() == n) { result.add(new ArrayList<>(current)); return; }
    for (int i = 0; i < n; i++) {
        if (!visited[i] && indegree[i] == 0) {
            visited[i] = true;
            current.add(i);
            for (int v : graph.get(i)) indegree[v]--;
            allTopoSort(current);
            for (int v : graph.get(i)) indegree[v]++;
            current.remove(current.size() - 1);
            visited[i] = false;
        }
    }
}

Dijkstra Template

int[] dijkstra(int n, List<List<int[]>> graph, int src) {
    int[] dist = new int[n];
    Arrays.fill(dist, Integer.MAX_VALUE);
    dist[src] = 0;

    // {distance, node}
    PriorityQueue<int[]> pq = new PriorityQueue<>((a,b) -> a[0]-b[0]);
    pq.offer(new int[]{0, src});

    while (!pq.isEmpty()) {
        int[] curr = pq.poll();
        int d = curr[0], u = curr[1];
        if (d > dist[u]) continue;  // stale entry
        for (int[] edge : graph.get(u)) {
            int v = edge[0], w = edge[1];
            if (dist[u] + w < dist[v]) {
                dist[v] = dist[u] + w;
                pq.offer(new int[]{dist[v], v});
            }
        }
    }
    return dist;
}

Bellman-Ford (handles negative edges, K-stop limit)

// LC 787 Cheapest Flights Within K Stops
int[] dist = new int[n];
Arrays.fill(dist, Integer.MAX_VALUE);
dist[src] = 0;

for (int i = 0; i <= k; i++) {
    int[] prev = dist.clone();  // prevent using current round
    for (int[] edge : edges) {
        int u = edge[0], v = edge[1], w = edge[2];
        if (prev[u] != Integer.MAX_VALUE)
            dist[v] = Math.min(dist[v], prev[u] + w);
    }
}

Floyd-Warshall (All-Pairs Shortest Paths)

Computes shortest paths between ALL pairs of nodes. O(V³) — only use when V ≤ 400 and you need all-pairs distances.

Key idea: "Can going through node k improve the shortest path from i to j?" Try every intermediate node k in the outer loop. The loop order MUST be k → i → j (not i → j → k).

// Floyd-Warshall: dist[i][j] = shortest path from i to j
int INF = Integer.MAX_VALUE / 2;
int[][] dist = new int[n][n];
for (int[] row : dist) Arrays.fill(row, INF);
for (int i = 0; i < n; i++) dist[i][i] = 0;
for (int[] e : edges) dist[e[0]][e[1]] = e[2]; // or min if multiple edges

// THE core triple loop — k MUST be outermost
for (int k = 0; k < n; k++)
    for (int i = 0; i < n; i++)
        for (int j = 0; j < n; j++)
            dist[i][j] = Math.min(dist[i][j], dist[i][k] + dist[k][j]);

Common bug: Using Integer.MAX_VALUE directly causes overflow when adding dist[i][k] + dist[k][j]. Use MAX_VALUE / 2 or check for INF before adding.

When to Use Floyd vs Dijkstra

Floyd Variants

  • Transitive closure: Replace min with OR, weights with boolean reachability. reach[i][j] |= reach[i][k] && reach[k][j]
  • Minimax path: Replace min(dist[i][j], dist[i][k] + dist[k][j]) with min(dist[i][j], max(dist[i][k], dist[k][j]))
  • Count shortest paths: Maintain parallel count[i][j] array
  • Path reconstruction: Store next[i][j] = k when improving, trace recursively

Union-Find (Disjoint Set Union)

Template with Path Compression + Union by Rank

class UnionFind {
    int[] parent, rank;
    int components;

    UnionFind(int n) {
        parent = new int[n];
        rank = new int[n];
        components = n;
        for (int i = 0; i < n; i++) parent[i] = i;
    }

    int find(int x) {
        if (parent[x] != x)
            parent[x] = find(parent[x]);  // path compression
        return parent[x];
    }

    boolean union(int x, int y) {
        int px = find(x), py = find(y);
        if (px == py) return false;
        if (rank[px] < rank[py]) { int t = px; px = py; py = t; }
        parent[py] = px;
        if (rank[px] == rank[py]) rank[px]++;
        components--;
        return true;
    }
}

When to Use

Dynamic connectivity: "are x and y connected?"
Count components as edges are added
Detect cycle in undirected graph: if union returns false
Kruskal's MST: sort edges + union-find

Minimum Spanning Tree

Kruskal's (Edge-based): O(E log E)

// Sort edges by weight, greedily add if no cycle
Arrays.sort(edges, (a, b) -> a[2] - b[2]);
UnionFind uf = new UnionFind(n);
int mstWeight = 0, edgeCount = 0;

for (int[] edge : edges) {
    if (uf.union(edge[0], edge[1])) {
        mstWeight += edge[2];
        if (++edgeCount == n - 1) break;
    }
}

Prim's (Vertex-based): O((V+E) log V)

// Start from any vertex, greedily pick cheapest edge to new vertex
PriorityQueue<int[]> pq = new PriorityQueue<>((a,b) -> a[1]-b[1]);
boolean[] inMST = new boolean[n];
pq.offer(new int[]{0, 0});  // {node, weight}

while (!pq.isEmpty()) {
    int[] curr = pq.poll();
    int u = curr[0], w = curr[1];
    if (inMST[u]) continue;
    inMST[u] = true;
    mstWeight += w;
    for (int[] edge : graph.get(u))
        if (!inMST[edge[0]])
            pq.offer(new int[]{edge[0], edge[1]});
}

Cycle Detection

Directed Graph: DFS with 3-color marking

// WHITE=0 (unvisited), GRAY=1 (in current path), BLACK=2 (done)
int[] color = new int[n];

boolean hasCycle(int u) {
    color[u] = 1;  // GRAY
    for (int v : graph.get(u)) {
        if (color[v] == 1) return true;  // back edge!
        if (color[v] == 0 && hasCycle(v)) return true;
    }
    color[u] = 2;  // BLACK
    return false;
}

Undirected Graph: Union-Find or DFS with parent tracking

// Union-Find: if union(u, v) returns false, edge creates cycle
for (int[] edge : edges) {
    if (!uf.union(edge[0], edge[1]))
        return edge;  // this edge creates cycle
}

Bipartite Graph

A graph whose vertices can be divided into two disjoint sets such that every edge connects a vertex in one set to a vertex in the other.

Check Bipartite: BFS 2-coloring

boolean isBipartite(int[][] graph) {
    int n = graph.length;
    int[] color = new int[n];  // 0=unvisited, 1=red, -1=blue

    for (int i = 0; i < n; i++) {
        if (color[i] != 0) continue;
        Queue<Integer> queue = new LinkedList<>();
        queue.offer(i);
        color[i] = 1;
        while (!queue.isEmpty()) {
            int u = queue.poll();
            for (int v : graph[u]) {
                if (color[v] == 0) {
                    color[v] = -color[u];
                    queue.offer(v);
                } else if (color[v] == color[u]) {
                    return false;  // odd cycle
                }
            }
        }
    }
    return true;
}

Key Insight

A graph is bipartite if and only if it contains no odd-length cycle. 2-coloring BFS is the standard check.

Strongly Connected Components & Bridges

Tarjan's Algorithm for Bridges (Cut Edges)

int timer = 0;
int[] disc, low;
List<int[]> bridges = new ArrayList<>();

void dfs(int u, int parent) {
    disc[u] = low[u] = timer++;
    for (int v : graph.get(u)) {
        if (v == parent) continue;
        if (disc[v] == -1) {
            dfs(v, u);
            low[u] = Math.min(low[u], low[v]);
            if (low[v] > disc[u])
                bridges.add(new int[]{u, v});  // bridge!
        } else {
            low[u] = Math.min(low[u], disc[v]);
        }
    }
}

Kosaraju's SCC (2-pass DFS)

Intuition: If we process nodes in reverse finish-time order on the reversed graph, each DFS tree we encounter is exactly one SCC. This works because a node finished late in pass 1 is a "root" of its SCC — reversing edges ensures its SCC members are only reachable from it, not from other SCCs.

// Kosaraju's Algorithm — returns list of SCCs
List<List<Integer>> kosaraju(int n, List<List<Integer>> graph) {
    // Pass 1: DFS on original graph, record finish order
    boolean[] visited = new boolean[n];
    Deque<Integer> finishOrder = new ArrayDeque<>();

    for (int i = 0; i < n; i++)
        if (!visited[i]) dfs1(i, graph, visited, finishOrder);

    // Build reversed graph
    List<List<Integer>> rev = new ArrayList<>();
    for (int i = 0; i < n; i++) rev.add(new ArrayList<>());
    for (int u = 0; u < n; u++)
        for (int v : graph.get(u)) rev.get(v).add(u);

    // Pass 2: DFS on reversed graph in reverse finish order
    Arrays.fill(visited, false);
    List<List<Integer>> sccs = new ArrayList<>();
    while (!finishOrder.isEmpty()) {
        int node = finishOrder.pop();
        if (!visited[node]) {
            List<Integer> scc = new ArrayList<>();
            dfs2(node, rev, visited, scc);
            sccs.add(scc);
        }
    }
    return sccs;
}

void dfs1(int u, List<List<Integer>> g, boolean[] vis, Deque<Integer> order) {
    vis[u] = true;
    for (int v : g.get(u)) if (!vis[v]) dfs1(v, g, vis, order);
    order.push(u); // record finish time
}

void dfs2(int u, List<List<Integer>> g, boolean[] vis, List<Integer> scc) {
    vis[u] = true;
    scc.add(u);
    for (int v : g.get(u)) if (!vis[v]) dfs2(v, g, vis, scc);
}

Tarjan's SCC (single-pass, stack-based)

Intuition: Maintain a stack of "currently in progress" nodes. A node is the root of its SCC if low[u] == disc[u] — meaning no node in its subtree can reach anything earlier. Pop everything above it from the stack = one SCC.

// Tarjan's SCC Algorithm
int timer = 0;
int[] disc, low, comp; // comp[u] = which SCC node u belongs to
boolean[] onStack;
Deque<Integer> stack = new ArrayDeque<>();
int numSCC = 0;

void tarjanSCC(int u) {
    disc[u] = low[u] = timer++;
    stack.push(u);
    onStack[u] = true;

    for (int v : graph.get(u)) {
        if (disc[v] == -1) {
            tarjanSCC(v);
            low[u] = Math.min(low[u], low[v]);
        } else if (onStack[v]) {
            low[u] = Math.min(low[u], disc[v]);
        }
    }

    // If u is root of an SCC
    if (low[u] == disc[u]) {
        while (true) {
            int v = stack.pop();
            onStack[v] = false;
            comp[v] = numSCC;
            if (v == u) break;
        }
        numSCC++;
    }
}

Euler Path / Circuit

What is it?

An Euler path visits every EDGE exactly once. An Euler circuit is an Euler path that starts and ends at the same vertex. (Contrast with Hamiltonian path which visits every VERTEX once — that's NP-hard.)

Existence Conditions

Graph typeEuler Circuit exists ifEuler Path exists if
UndirectedAll vertices have even degree AND graph is connectedExactly 0 or 2 vertices have odd degree
DirectedEvery vertex has in-degree = out-degree AND graph is strongly connectedAt most one vertex has out-in=1 (start), at most one has in-out=1 (end)

Hierholzer's Algorithm (Euler Circuit/Path)

Key idea: DFS but remove edges as you traverse them. When you get stuck (no more edges from current node), add current node to the result and backtrack. The result (reversed) is the Euler path. This is NOT regular DFS — you must delete edges to avoid reuse.

// Hierholzer's for DIRECTED graph (LC 332 Reconstruct Itinerary)
Map<String, PriorityQueue<String>> graph = new HashMap<>();
List<String> result = new ArrayList<>();

void dfs(String node) {
    PriorityQueue<String> neighbors = graph.get(node);
    while (neighbors != null && !neighbors.isEmpty()) {
        String next = neighbors.poll();  // remove edge (use it up)
        dfs(next);
    }
    result.add(node);  // add to front when "stuck"
}

// Call: dfs(startNode); Collections.reverse(result);

// For UNDIRECTED graph — mark edges as used (need edge IDs)
void hierholzerUndirected(int start) {
    Deque<Integer> stack = new ArrayDeque<>();
    List<Integer> path = new ArrayList<>();
    stack.push(start);
    while (!stack.isEmpty()) {
        int u = stack.peek();
        if (hasUnusedEdge(u)) {
            int[] edge = getNextUnusedEdge(u);
            markUsed(edge);
            stack.push(edge[1]); // go to other endpoint
        } else {
            path.add(stack.pop()); // backtrack: add to result
        }
    }
    Collections.reverse(path);
}

Common interview disguise: "Reconstruct Itinerary" (LC 332) = Euler path on directed multigraph. "Cracking the Safe" (LC 753) = Euler circuit on de Bruijn graph. Recognize these as Euler problems and Hierholzer's follows naturally.

Real-world uses

  • Chinese Postman Problem (mail delivery visiting every street)
  • DNA sequencing (Euler path on de Bruijn graph of k-mers)
  • Drawing puzzles ("draw without lifting your pen")

A* Search

What is it?

Informed search that uses a heuristic to guide exploration. Like Dijkstra, but instead of expanding the node with smallest g(n) (cost so far), it expands the node with smallest f(n) = g(n) + h(n) where h(n) estimates remaining cost to goal.

When is it useful?

  • Games: pathfinding on maps (characters navigating obstacles)
  • Grid problems: when BFS is too slow and you have a natural distance heuristic
  • State-space search: sliding puzzles, Rubik's cube
  • Interviews: rare, but LC 1197 (Minimum Knight Moves) can use it

Key Property: Admissibility

h(n) must be admissible: it never OVERestimates the true remaining cost. If admissible, A* finds the optimal path. Common heuristics: Manhattan distance (grid, 4-directional), Euclidean distance (continuous), Chebyshev distance (8-directional).

// A* Search Template
int aStar(int[][] grid, int[] start, int[] goal) {
    int n = grid.length, m = grid[0].length;
    int[][] dist = new int[n][m];
    for (int[] row : dist) Arrays.fill(row, Integer.MAX_VALUE);
    dist[start[0]][start[1]] = 0;

    // PQ ordered by f = g + h
    PriorityQueue<int[]> pq = new PriorityQueue<>((a, b) -> a[2] - b[2]);
    int h = heuristic(start, goal);
    pq.offer(new int[]{start[0], start[1], h}); // {row, col, f}

    int[][] dirs = {{0,1},{0,-1},{1,0},{-1,0}};
    while (!pq.isEmpty()) {
        int[] curr = pq.poll();
        int r = curr[0], c = curr[1];
        if (r == goal[0] && c == goal[1]) return dist[r][c];
        if (curr[2] > dist[r][c] + heuristic(new int[]{r,c}, goal)) continue;

        for (int[] d : dirs) {
            int nr = r + d[0], nc = c + d[1];
            if (nr < 0 || nr >= n || nc < 0 || nc >= m) continue;
            int newDist = dist[r][c] + grid[nr][nc]; // edge cost
            if (newDist < dist[nr][nc]) {
                dist[nr][nc] = newDist;
                pq.offer(new int[]{nr, nc, newDist + heuristic(new int[]{nr,nc}, goal)});
            }
        }
    }
    return -1;
}

int heuristic(int[] a, int[] b) {
    return Math.abs(a[0] - b[0]) + Math.abs(a[1] - b[1]); // Manhattan
}

A* vs Dijkstra vs BFS

AlgorithmWhen to useExplores
BFSUnweighted, guaranteed shortestAll nodes at distance d before d+1
DijkstraWeighted, no heuristic availableExpands by cheapest g(n)
A*Weighted, good heuristic availableExpands by cheapest g(n) + h(n) → fewer nodes explored

Interview context: If asked "how to speed up Dijkstra on a grid?", mention A* with Manhattan heuristic. In practice, A* explores far fewer nodes when the heuristic is tight. For LC problems, BFS/Dijkstra usually suffice — A* is more of a system design / game dev topic.
