The Dummy Head Technique

The single most useful trick for linked list problems. A dummy node before the actual head eliminates special cases for head insertion/deletion.

// Without dummy head: must handle head specially
if (head.val == target) {
    head = head.next;  // special case
}

// With dummy head: uniform handling
ListNode dummy = new ListNode(0);
dummy.next = head;
ListNode prev = dummy;
while (prev.next != null) {
    if (prev.next.val == target) {
        prev.next = prev.next.next;
    } else {
        prev = prev.next;
    }
}
return dummy.next;

Key Insight: In interviews, linked list problems test pointer manipulation. Off-by-one errors and null pointer exceptions are the main bugs. Always trace through your code with a 1-node and 2-node list before submitting.

Standard ListNode Definition

public class ListNode {
    int val;
    ListNode next;
    ListNode(int val) { this.val = val; }
    ListNode(int val, ListNode next) { this.val = val; this.next = next; }
}

Reversal — The Most Important Operation

Google WILL ask you to do both recursive and iterative reversal. Be ready to switch between them in under 2 minutes. This is the foundation for 50% of linked list problems.

Iterative Reversal (LC 206)

The classic three-pointer approach: prev, curr, next.

// Iterative: O(n) time, O(1) space
public ListNode reverseList(ListNode head) {
    ListNode prev = null, curr = head;
    while (curr != null) {
        ListNode next = curr.next;  // save next
        curr.next = prev;           // reverse pointer
        prev = curr;                // advance prev
        curr = next;                // advance curr
    }
    return prev;  // prev is the new head
}

Recursive Reversal (LC 206)

Elegant but uses O(n) stack space. The key insight: after recursive call returns, head.next is the tail of the reversed sublist.

// Recursive: O(n) time, O(n) space (call stack)
public ListNode reverseList(ListNode head) {
    if (head == null || head.next == null) return head;
    ListNode newHead = reverseList(head.next);
    head.next.next = head;  // head.next points back to head
    head.next = null;        // head becomes tail
    return newHead;
}

Why does this work? After reverseList(head.next), everything from head.next onward is reversed. head.next is now the LAST node of that reversed sublist. So head.next.next = head appends head to the end.

Reverse Between Positions (LC 92)

Reverse only the sublist from position left to right. Key: navigate to the node before left, then reverse exactly right - left links.

// Reverse sublist [left, right]: O(n) time, O(1) space
public ListNode reverseBetween(ListNode head, int left, int right) {
    ListNode dummy = new ListNode(0, head);
    ListNode prev = dummy;

    // Move prev to the node before position 'left'
    for (int i = 1; i < left; i++) {
        prev = prev.next;
    }

    // curr is the first node to reverse
    ListNode curr = prev.next;

    // Reverse (right - left) links using "insert at front" technique
    for (int i = 0; i < right - left; i++) {
        ListNode next = curr.next;
        curr.next = next.next;
        next.next = prev.next;
        prev.next = next;
    }

    return dummy.next;
}

Trick: The "insert at front" technique repeatedly takes the next node after curr and inserts it right after prev. This reverses the sublist in-place without needing to reconnect boundaries afterwards.

Reverse in K-Groups (LC 25)

The hardest reversal variant. Strategy: count K nodes, reverse them, recursively/iteratively handle the rest.

// Iterative approach: O(n) time, O(1) space
public ListNode reverseKGroup(ListNode head, int k) {
    ListNode dummy = new ListNode(0, head);
    ListNode groupPrev = dummy;

    while (true) {
        // Check if there are k nodes remaining
        ListNode kth = getKth(groupPrev, k);
        if (kth == null) break;

        ListNode groupNext = kth.next;

        // Reverse the group
        ListNode prev = groupNext, curr = groupPrev.next;
        while (curr != groupNext) {
            ListNode next = curr.next;
            curr.next = prev;
            prev = curr;
            curr = next;
        }

        // Connect with previous part
        ListNode tmp = groupPrev.next;  // this was first, now last in group
        groupPrev.next = kth;           // kth is now first in group
        groupPrev = tmp;                // move to end of reversed group
    }

    return dummy.next;
}

private ListNode getKth(ListNode curr, int k) {
    while (curr != null && k > 0) {
        curr = curr.next;
        k--;
    }
    return curr;
}

Recursive version of Reverse K-Group

// Recursive: O(n) time, O(n/k) space
public ListNode reverseKGroup(ListNode head, int k) {
    // Count k nodes
    ListNode curr = head;
    int count = 0;
    while (curr != null && count < k) {
        curr = curr.next;
        count++;
    }
    if (count < k) return head;  // fewer than k, don't reverse

    // Reverse first k nodes
    ListNode prev = reverseKGroup(curr, k);  // recurse on rest
    while (count-- > 0) {
        ListNode next = head.next;
        head.next = prev;
        prev = head;
        head = next;
    }
    return prev;
}

Fast & Slow Pointers (Floyd's Technique)

Two pointers moving at different speeds reveal structural properties of the list: cycles, midpoints, and distances from end.

Cycle Detection (LC 141)

Floyd's Tortoise and Hare: slow moves 1 step, fast moves 2 steps. If there's a cycle, they meet.

// O(n) time, O(1) space
public boolean hasCycle(ListNode head) {
    ListNode slow = head, fast = head;
    while (fast != null && fast.next != null) {
        slow = slow.next;
        fast = fast.next.next;
        if (slow == fast) return true;
    }
    return false;
}

Find Cycle Start (LC 142)

After detecting a cycle, reset one pointer to head. Move both at speed 1 — they meet at the cycle start.

Math Proof: Let distance from head to cycle start = a, cycle start to meeting point = b, meeting point back to cycle start = c. Slow travels a + b. Fast travels a + b + (b + c) * k = 2(a + b). So a + b = k(b + c), meaning a = (k-1)(b+c) + c. Thus moving a steps from meeting point lands at cycle start!

public ListNode detectCycle(ListNode head) {
    ListNode slow = head, fast = head;
    while (fast != null && fast.next != null) {
        slow = slow.next;
        fast = fast.next.next;
        if (slow == fast) {
            ListNode entry = head;
            while (entry != slow) {
                entry = entry.next;
                slow = slow.next;
            }
            return entry;
        }
    }
    return null;
}

Find Middle (LC 876)

When fast reaches the end, slow is at the middle. For even-length lists, this gives the second middle node.

public ListNode middleNode(ListNode head) {
    ListNode slow = head, fast = head;
    while (fast != null && fast.next != null) {
        slow = slow.next;
        fast = fast.next.next;
    }
    return slow;  // second middle for even length
}

// To get FIRST middle (useful for merge sort split):
public ListNode firstMiddle(ListNode head) {
    ListNode slow = head, fast = head.next;  // fast starts one ahead
    while (fast != null && fast.next != null) {
        slow = slow.next;
        fast = fast.next.next;
    }
    return slow;  // first middle for even length
}

Subtle difference: fast = head gives second middle. fast = head.next gives first middle. This matters for merge sort where you need to split exactly in half.

Remove Nth from End (LC 19)

Fast pointer moves n steps ahead, then both move together. When fast reaches end, slow is at the (n-1)th from end.

public ListNode removeNthFromEnd(ListNode head, int n) {
    ListNode dummy = new ListNode(0, head);
    ListNode fast = dummy, slow = dummy;

    // Advance fast by n+1 steps
    for (int i = 0; i <= n; i++) {
        fast = fast.next;
    }

    // Move both until fast reaches null
    while (fast != null) {
        slow = slow.next;
        fast = fast.next;
    }

    slow.next = slow.next.next;  // delete the nth node
    return dummy.next;
}

Palindrome Check (LC 234)

Find middle, reverse second half, compare both halves. O(1) space (excluding the reversal modification).

public boolean isPalindrome(ListNode head) {
    // 1. Find middle
    ListNode slow = head, fast = head;
    while (fast != null && fast.next != null) {
        slow = slow.next;
        fast = fast.next.next;
    }

    // 2. Reverse second half
    ListNode prev = null;
    while (slow != null) {
        ListNode next = slow.next;
        slow.next = prev;
        prev = slow;
        slow = next;
    }

    // 3. Compare
    ListNode left = head, right = prev;
    while (right != null) {
        if (left.val != right.val) return false;
        left = left.next;
        right = right.next;
    }
    return true;
}

Merge Operations

Merge Two Sorted Lists (LC 21)

The building block for merge sort and merge K lists. Use a dummy head for clean code.

// Iterative with dummy: O(n + m) time, O(1) space
public ListNode mergeTwoLists(ListNode l1, ListNode l2) {
    ListNode dummy = new ListNode(0);
    ListNode curr = dummy;

    while (l1 != null && l2 != null) {
        if (l1.val <= l2.val) {
            curr.next = l1;
            l1 = l1.next;
        } else {
            curr.next = l2;
            l2 = l2.next;
        }
        curr = curr.next;
    }
    curr.next = (l1 != null) ? l1 : l2;
    return dummy.next;
}

Merge K Sorted Lists (LC 23)

Approach 1: Min-Heap (Priority Queue)

// O(N log k) time, O(k) space where N = total nodes, k = num lists
public ListNode mergeKLists(ListNode[] lists) {
    PriorityQueue<ListNode> pq = new PriorityQueue<>(
        (a, b) -> a.val - b.val
    );

    for (ListNode node : lists) {
        if (node != null) pq.offer(node);
    }

    ListNode dummy = new ListNode(0);
    ListNode curr = dummy;

    while (!pq.isEmpty()) {
        ListNode node = pq.poll();
        curr.next = node;
        curr = curr.next;
        if (node.next != null) pq.offer(node.next);
    }

    return dummy.next;
}

Approach 2: Divide and Conquer

// O(N log k) time, O(log k) space (recursion)
public ListNode mergeKLists(ListNode[] lists) {
    if (lists.length == 0) return null;
    return merge(lists, 0, lists.length - 1);
}

private ListNode merge(ListNode[] lists, int lo, int hi) {
    if (lo == hi) return lists[lo];
    int mid = lo + (hi - lo) / 2;
    ListNode left = merge(lists, lo, mid);
    ListNode right = merge(lists, mid + 1, hi);
    return mergeTwoLists(left, right);
}

Which to choose? Both are O(N log k). Heap approach is simpler to code but uses a PQ. Divide-and-conquer reuses the merge-two-lists subroutine and is often preferred in interviews because it shows you understand the merge sort pattern.

Sort List (LC 148)

Merge sort on a linked list. Find middle, split, recursively sort, then merge.

// O(n log n) time, O(log n) space for recursion
public ListNode sortList(ListNode head) {
    if (head == null || head.next == null) return head;

    // Find middle (use first middle to split evenly)
    ListNode slow = head, fast = head.next;
    while (fast != null && fast.next != null) {
        slow = slow.next;
        fast = fast.next.next;
    }

    ListNode mid = slow.next;
    slow.next = null;  // cut the list

    ListNode left = sortList(head);
    ListNode right = sortList(mid);
    return mergeTwoLists(left, right);
}

Insertion Sort List (LC 147)

// O(n^2) time, O(1) space
public ListNode insertionSortList(ListNode head) {
    ListNode dummy = new ListNode(0);
    ListNode curr = head;

    while (curr != null) {
        ListNode next = curr.next;

        // Find insertion position
        ListNode prev = dummy;
        while (prev.next != null && prev.next.val < curr.val) {
            prev = prev.next;
        }

        // Insert curr after prev
        curr.next = prev.next;
        prev.next = curr;
        curr = next;
    }

    return dummy.next;
}

Intersection & Combination

Intersection of Two Lists (LC 160)

Two-pointer trick: when one pointer reaches the end, redirect it to the other list's head. They meet at the intersection after traversing equal distances.

// O(n + m) time, O(1) space
public ListNode getIntersectionNode(ListNode headA, ListNode headB) {
    ListNode a = headA, b = headB;
    while (a != b) {
        a = (a != null) ? a.next : headB;
        b = (b != null) ? b.next : headA;
    }
    return a;  // either intersection node or null
}

Why does this work? Pointer A travels: lenA + lenB - common. Pointer B travels: lenB + lenA - common. Same total distance, so they arrive at the intersection simultaneously.

Add Two Numbers (LC 2)

Numbers stored in reverse order (least significant digit first). Process digit by digit with carry.

public ListNode addTwoNumbers(ListNode l1, ListNode l2) {
    ListNode dummy = new ListNode(0);
    ListNode curr = dummy;
    int carry = 0;

    while (l1 != null || l2 != null || carry != 0) {
        int sum = carry;
        if (l1 != null) { sum += l1.val; l1 = l1.next; }
        if (l2 != null) { sum += l2.val; l2 = l2.next; }
        carry = sum / 10;
        curr.next = new ListNode(sum % 10);
        curr = curr.next;
    }

    return dummy.next;
}

Add Two Numbers II (LC 445)

Numbers stored in forward order (most significant first). Use stacks to process from the end.

public ListNode addTwoNumbers(ListNode l1, ListNode l2) {
    Deque<Integer> s1 = new ArrayDeque<>();
    Deque<Integer> s2 = new ArrayDeque<>();

    while (l1 != null) { s1.push(l1.val); l1 = l1.next; }
    while (l2 != null) { s2.push(l2.val); l2 = l2.next; }

    ListNode head = null;
    int carry = 0;

    while (!s1.isEmpty() || !s2.isEmpty() || carry != 0) {
        int sum = carry;
        if (!s1.isEmpty()) sum += s1.pop();
        if (!s2.isEmpty()) sum += s2.pop();
        carry = sum / 10;
        ListNode node = new ListNode(sum % 10);
        node.next = head;
        head = node;  // build result in reverse
    }

    return head;
}

Copy List with Random Pointer (LC 138)

Approach 1: HashMap

// O(n) time, O(n) space
public Node copyRandomList(Node head) {
    Map<Node, Node> map = new HashMap<>();
    Node curr = head;

    // First pass: create all nodes
    while (curr != null) {
        map.put(curr, new Node(curr.val));
        curr = curr.next;
    }

    // Second pass: wire next and random pointers
    curr = head;
    while (curr != null) {
        map.get(curr).next = map.get(curr.next);
        map.get(curr).random = map.get(curr.random);
        curr = curr.next;
    }

    return map.get(head);
}

Approach 2: Interleave (O(1) space)

// O(n) time, O(1) space
public Node copyRandomList(Node head) {
    if (head == null) return null;

    // Step 1: Interleave copies: A->A'->B->B'->C->C'
    Node curr = head;
    while (curr != null) {
        Node copy = new Node(curr.val);
        copy.next = curr.next;
        curr.next = copy;
        curr = copy.next;
    }

    // Step 2: Set random pointers for copies
    curr = head;
    while (curr != null) {
        if (curr.random != null) {
            curr.next.random = curr.random.next;
        }
        curr = curr.next.next;
    }

    // Step 3: Separate the two lists
    Node newHead = head.next;
    curr = head;
    while (curr != null) {
        Node copy = curr.next;
        curr.next = copy.next;
        copy.next = (copy.next != null) ? copy.next.next : null;
        curr = curr.next;
    }

    return newHead;
}

Flatten a Multilevel Doubly Linked List (LC 430)

// DFS approach: O(n) time, O(depth) space
public Node flatten(Node head) {
    Node curr = head;
    while (curr != null) {
        if (curr.child != null) {
            Node next = curr.next;
            Node child = flatten(curr.child);

            curr.next = child;
            child.prev = curr;
            curr.child = null;

            // Find tail of flattened child
            Node tail = child;
            while (tail.next != null) tail = tail.next;

            tail.next = next;
            if (next != null) next.prev = tail;
        }
        curr = curr.next;
    }
    return head;
}

Modification & Rearrangement

Remove Duplicates from Sorted List (LC 83)

// Keep one of each duplicate: O(n) time, O(1) space
public ListNode deleteDuplicates(ListNode head) {
    ListNode curr = head;
    while (curr != null && curr.next != null) {
        if (curr.val == curr.next.val) {
            curr.next = curr.next.next;
        } else {
            curr = curr.next;
        }
    }
    return head;
}

Remove All Duplicates (LC 82)

// Remove all nodes with duplicate values: O(n) time, O(1) space
public ListNode deleteDuplicates(ListNode head) {
    ListNode dummy = new ListNode(0, head);
    ListNode prev = dummy;

    while (prev.next != null) {
        ListNode curr = prev.next;
        while (curr.next != null && curr.next.val == curr.val) {
            curr = curr.next;
        }
        if (curr != prev.next) {
            // duplicates found, skip all of them
            prev.next = curr.next;
        } else {
            prev = prev.next;
        }
    }

    return dummy.next;
}

Partition List (LC 86)

Create two separate lists (less-than and greater-or-equal), then join them.

public ListNode partition(ListNode head, int x) {
    ListNode lessHead = new ListNode(0);
    ListNode greaterHead = new ListNode(0);
    ListNode less = lessHead, greater = greaterHead;

    while (head != null) {
        if (head.val < x) {
            less.next = head;
            less = less.next;
        } else {
            greater.next = head;
            greater = greater.next;
        }
        head = head.next;
    }

    greater.next = null;  // important: terminate!
    less.next = greaterHead.next;
    return lessHead.next;
}

Common bug: Forgetting greater.next = null creates a cycle in the resulting list!

Reorder List (LC 143)

Transform L0 → L1 → ... → Ln into L0 → Ln → L1 → Ln-1 → ...

Three steps: find middle, reverse second half, interleave.

public void reorderList(ListNode head) {
    if (head == null || head.next == null) return;

    // 1. Find middle
    ListNode slow = head, fast = head;
    while (fast.next != null && fast.next.next != null) {
        slow = slow.next;
        fast = fast.next.next;
    }

    // 2. Reverse second half
    ListNode second = slow.next;
    slow.next = null;
    ListNode prev = null;
    while (second != null) {
        ListNode next = second.next;
        second.next = prev;
        prev = second;
        second = next;
    }

    // 3. Interleave
    ListNode first = head;
    second = prev;
    while (second != null) {
        ListNode tmp1 = first.next, tmp2 = second.next;
        first.next = second;
        second.next = tmp1;
        first = tmp1;
        second = tmp2;
    }
}

Swap Nodes in Pairs (LC 24)

Iterative

public ListNode swapPairs(ListNode head) {
    ListNode dummy = new ListNode(0, head);
    ListNode prev = dummy;

    while (prev.next != null && prev.next.next != null) {
        ListNode first = prev.next;
        ListNode second = first.next;

        first.next = second.next;
        second.next = first;
        prev.next = second;
        prev = first;
    }

    return dummy.next;
}

Recursive

public ListNode swapPairs(ListNode head) {
    if (head == null || head.next == null) return head;
    ListNode second = head.next;
    head.next = swapPairs(second.next);
    second.next = head;
    return second;
}

Rotate List (LC 61)

Connect tail to head (make circular), then break at the right position.

public ListNode rotateRight(ListNode head, int k) {
    if (head == null || head.next == null) return head;

    // Find length and tail
    int len = 1;
    ListNode tail = head;
    while (tail.next != null) { tail = tail.next; len++; }

    k %= len;
    if (k == 0) return head;

    // Make circular
    tail.next = head;

    // Move to the new tail (len - k - 1 steps from head, or len - k from tail)
    ListNode newTail = head;
    for (int i = 0; i < len - k - 1; i++) {
        newTail = newTail.next;
    }

    ListNode newHead = newTail.next;
    newTail.next = null;
    return newHead;
}

Odd Even Linked List (LC 328)

public ListNode oddEvenList(ListNode head) {
    if (head == null) return null;
    ListNode odd = head, even = head.next, evenHead = even;

    while (even != null && even.next != null) {
        odd.next = even.next;
        odd = odd.next;
        even.next = odd.next;
        even = even.next;
    }

    odd.next = evenHead;
    return head;
}

Design Problems

LRU Cache (LC 146)

The most important design problem using a linked list. Doubly linked list + HashMap gives O(1) for both get and put.

Google asks this frequently. You must be able to code the full implementation in under 15 minutes without bugs.

class LRUCache {
    private int capacity;
    private Map<Integer, Node> map;
    private Node head, tail;  // dummy sentinels

    class Node {
        int key, val;
        Node prev, next;
        Node(int k, int v) { key = k; val = v; }
    }

    public LRUCache(int capacity) {
        this.capacity = capacity;
        map = new HashMap<>();
        head = new Node(0, 0);
        tail = new Node(0, 0);
        head.next = tail;
        tail.prev = head;
    }

    public int get(int key) {
        if (!map.containsKey(key)) return -1;
        Node node = map.get(key);
        remove(node);
        insertAtHead(node);
        return node.val;
    }

    public void put(int key, int value) {
        if (map.containsKey(key)) {
            remove(map.get(key));
        }
        Node node = new Node(key, value);
        map.put(key, node);
        insertAtHead(node);

        if (map.size() > capacity) {
            Node lru = tail.prev;
            remove(lru);
            map.remove(lru.key);
        }
    }

    private void remove(Node node) {
        node.prev.next = node.next;
        node.next.prev = node.prev;
    }

    private void insertAtHead(Node node) {
        node.next = head.next;
        node.prev = head;
        head.next.prev = node;
        head.next = node;
    }
}

Key Design Decision: Head = most recently used, Tail = least recently used. Every access moves the node to head. Eviction removes from tail. Both operations are O(1) with doubly linked list.

LFU Cache (LC 460)

Extension of LRU: evict the least frequently used. Use a frequency map where each frequency maps to an ordered set (or doubly linked list) of nodes. Track the minimum frequency.

Structure: Map<Integer, Node> keyMap + Map<Integer, DoublyLinkedList> freqMap + int minFreq. On access, move node from freq F list to freq F+1 list. If freq F list becomes empty and F == minFreq, increment minFreq.

Design Linked List (LC 707)

class MyLinkedList {
    private Node head, tail;
    private int size;

    class Node {
        int val;
        Node prev, next;
        Node(int v) { val = v; }
    }

    public MyLinkedList() {
        head = new Node(0);
        tail = new Node(0);
        head.next = tail;
        tail.prev = head;
        size = 0;
    }

    public int get(int index) {
        if (index < 0 || index >= size) return -1;
        Node curr = head.next;
        for (int i = 0; i < index; i++) curr = curr.next;
        return curr.val;
    }

    public void addAtIndex(int index, int val) {
        if (index > size) return;
        if (index < 0) index = 0;
        Node pred = head;
        for (int i = 0; i < index; i++) pred = pred.next;
        Node node = new Node(val);
        node.next = pred.next;
        node.prev = pred;
        pred.next.prev = node;
        pred.next = node;
        size++;
    }

    public void addAtHead(int val) { addAtIndex(0, val); }
    public void addAtTail(int val) { addAtIndex(size, val); }

    public void deleteAtIndex(int index) {
        if (index < 0 || index >= size) return;
        Node pred = head;
        for (int i = 0; i < index; i++) pred = pred.next;
        pred.next = pred.next.next;
        pred.next.prev = pred;
        size--;
    }
}

Flatten Nested List Iterator (LC 341)

class NestedIterator implements Iterator<Integer> {
    private Deque<Iterator<NestedInteger>> stack;
    private Integer nextVal;

    public NestedIterator(List<NestedInteger> nestedList) {
        stack = new ArrayDeque<>();
        stack.push(nestedList.iterator());
    }

    public boolean hasNext() {
        while (!stack.isEmpty()) {
            Iterator<NestedInteger> it = stack.peek();
            if (!it.hasNext()) {
                stack.pop();
                continue;
            }
            NestedInteger ni = it.next();
            if (ni.isInteger()) {
                nextVal = ni.getInteger();
                return true;
            }
            stack.push(ni.getList().iterator());
        }
        return false;
    }

    public Integer next() { return nextVal; }
}

Reverse Nodes in Even Length Groups (LC 2074)

Traverse the list in groups of increasing size (1, 2, 3, ...). Reverse the group only if its actual length is even.

public ListNode reverseEvenLengthGroups(ListNode head) {
    ListNode prev = head;
    int groupSize = 2;  // first group of 1 is never reversed

    while (prev.next != null) {
        // Count actual nodes in this group
        ListNode node = prev;
        int count = 0;
        for (int i = 0; i < groupSize && node.next != null; i++) {
            node = node.next;
            count++;
        }

        if (count % 2 == 0) {
            // Reverse 'count' nodes starting from prev.next
            ListNode curr = prev.next;
            for (int i = 0; i < count - 1; i++) {
                ListNode next = curr.next;
                curr.next = next.next;
                next.next = prev.next;
                prev.next = next;
            }
            prev = curr;
        } else {
            // Skip this group
            for (int i = 0; i < count; i++) {
                prev = prev.next;
            }
        }

        groupSize++;
    }

    return head;
}

Detect and Remove Loop

Combine Floyd's cycle detection with removal: once you find the cycle start, traverse to find the node whose next points to the cycle start, then set it to null.

public void removeLoop(ListNode head) {
    ListNode slow = head, fast = head;

    // Detect cycle
    while (fast != null && fast.next != null) {
        slow = slow.next;
        fast = fast.next.next;
        if (slow == fast) break;
    }

    if (fast == null || fast.next == null) return;  // no cycle

    // Find cycle start
    slow = head;
    if (slow == fast) {
        // Special case: cycle starts at head
        while (fast.next != slow) fast = fast.next;
    } else {
        while (slow.next != fast.next) {
            slow = slow.next;
            fast = fast.next;
        }
    }

    fast.next = null;  // remove the loop
}

Linked List in Binary Tree (LC 1367)

Check if a linked list corresponds to any downward path in a binary tree. Use DFS with a helper that matches from the current tree node.

public boolean isSubPath(ListNode head, TreeNode root) {
    if (root == null) return false;
    return matches(head, root) ||
           isSubPath(head, root.left) ||
           isSubPath(head, root.right);
}

private boolean matches(ListNode list, TreeNode tree) {
    if (list == null) return true;   // entire list matched
    if (tree == null) return false;  // tree ended before list
    if (list.val != tree.val) return false;
    return matches(list.next, tree.left) ||
           matches(list.next, tree.right);
}

Optimization: For very long lists, precompute KMP failure function on the list values to avoid re-scanning. But for interviews, the simple DFS approach is sufficient and expected.

Dummy Head Rules

When to use dummy head: (1) Deletion might affect head. (2) Insertion before head. (3) Merging lists. (4) Any time the first node needs the same treatment as others. When in doubt, use a dummy head — it costs nothing and prevents bugs.

Common Bugs to Avoid

  1. Forgetting to save next before overwriting. Classic in reversal: ListNode next = curr.next BEFORE curr.next = prev.
  2. Null pointer on empty list or single node. Always check head == null and head.next == null at the start.
  3. Not terminating a rearranged list. After partition or rotation, the last node's next might still point somewhere — set it to null!
  4. Off-by-one in counting. "Remove nth from end" — do you advance n or n+1 steps? Use dummy head to avoid this.
  5. Modifying the list while traversing. In "remove duplicates", make sure you don't skip nodes by advancing only in the else branch.

Google Follow-up Patterns

Common follow-ups:

  • "Can you do it in O(1) space?" → Usually means reverse-in-place instead of using a stack or extra data structure.
  • "Can you do it iteratively?" → After you show recursive. Convert using explicit stack or prev/curr pattern.
  • "What if the list is doubly linked?" → Usually makes the problem easier. Don't forget to update both prev and next pointers.
  • "Can you do it in one pass?" → Think fast/slow pointers or maintain a fixed gap between two pointers.
  • "What about thread safety?" → LRU Cache follow-up. Discuss ConcurrentHashMap + ReentrantLock on the list operations.

Drawing Pointer Diagrams

Always draw before you code. For each pointer manipulation, draw the state before and after. Label each step. This takes 1-2 minutes but saves 5-10 minutes of debugging. In virtual interviews, use the drawing tool or describe the state verbally to the interviewer.
