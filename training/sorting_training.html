Merge Sort Template (for counting problems)

void mergeSort(int[] arr, int lo, int hi) {
    if (lo >= hi) return;
    int mid = lo + (hi - lo) / 2;
    mergeSort(arr, lo, mid);
    mergeSort(arr, mid + 1, hi);
    merge(arr, lo, mid, hi);
}

void merge(int[] arr, int lo, int mid, int hi) {
    int[] temp = Arrays.copyOfRange(arr, lo, hi + 1);
    int i = 0, j = mid - lo + 1, k = lo;
    while (i <= mid - lo && j <= hi - lo) {
        if (temp[i] <= temp[j]) arr[k++] = temp[i++];
        else { arr[k++] = temp[j++]; count += (mid - lo - i + 1); }
    }
    while (i <= mid - lo) arr[k++] = temp[i++];
    while (j <= hi - lo) arr[k++] = temp[j++];
}

Quick Select: O(n) average Kth element

int quickSelect(int[] nums, int lo, int hi, int k) {
    int pivot = nums[hi], i = lo;
    for (int j = lo; j < hi; j++)
        if (nums[j] <= pivot) swap(nums, i++, j);
    swap(nums, i, hi);

    if (i == k) return nums[i];
    else if (i < k) return quickSelect(nums, i + 1, hi, k);
    else return quickSelect(nums, lo, i - 1, k);
}

Sorting as Preprocessing

Pattern: Sort then Two Pointers

// LC 15 3Sum: sort + fix one + two pointers on rest
// LC 16 3Sum Closest: same pattern, track closest
// LC 18 4Sum: two nested loops + two pointers
Arrays.sort(nums);
// Now duplicates are adjacent (easy skip) and two-pointer works

Pattern: Sort then Binary Search

// LC 300 LIS with patience sorting
// Maintain sorted "tails" array, binary search for insertion point
List<Integer> tails = new ArrayList<>();
for (int num : nums) {
    int pos = Collections.binarySearch(tails, num);
    if (pos < 0) pos = -(pos + 1);
    if (pos == tails.size()) tails.add(num);
    else tails.set(pos, num);
}
return tails.size();

Russian Doll Envelopes (LC 354): Sort by width, then LIS on height.
Meeting Rooms (LC 253): Sort by start, use heap for end times.
Job Scheduling (LC 1235): Sort by end, DP + binary search.

Custom Comparators

Multi-Key Sorting

// Sort by key1 ascending, then key2 descending
Arrays.sort(arr, (a, b) -> {
    if (a[0] != b[0]) return a[0] - b[0];
    return b[1] - a[1];  // descending
});

String Concatenation Comparator

// LC 179 Largest Number
Arrays.sort(strs, (a, b) -> (b + a).compareTo(a + b));
// "9" + "34" = "934" > "349" = "34" + "9", so "9" first

Event-Based Sorting (start/end events)

// Create events: +1 at start, -1 at end
int[][] events = new int[2 * n][2];
for (int i = 0; i < n; i++) {
    events[2*i] = new int[]{intervals[i][0], 1};   // start
    events[2*i+1] = new int[]{intervals[i][1], -1}; // end
}
Arrays.sort(events, (a, b) -> a[0] != b[0] ? a[0]-b[0] : a[1]-b[1]);

Counting Sort / Bucket Sort / Radix Sort

Counting Sort: O(n + k)

// When values are in a bounded range [0, k]
int[] count = new int[k + 1];
for (int num : nums) count[num]++;
int idx = 0;
for (int val = 0; val <= k; val++)
    while (count[val]-- > 0) nums[idx++] = val;

Bucket Sort: distribute into buckets, sort each

// LC 164 Maximum Gap: bucket sort guarantee
// If n elements in [min, max], max gap >= ceil((max-min)/(n-1))
// Use buckets of that size, max gap is between consecutive non-empty buckets
int bucketSize = Math.max(1, (max - min) / (n - 1));
int bucketCount = (max - min) / bucketSize + 1;
// Track min/max per bucket, answer = max(bucket[i].min - bucket[i-1].max)

Frequency Bucket Sort

// LC 347 Top K Frequent: bucket[freq] = elements with that frequency
// LC 451 Sort Characters By Frequency: same idea
List<Integer>[] buckets = new List[n + 1];
for (var entry : freqMap.entrySet()) {
    int freq = entry.getValue();
    if (buckets[freq] == null) buckets[freq] = new ArrayList<>();
    buckets[freq].add(entry.getKey());
}
// Iterate from high to low frequency

Merge Sort Applications

The merge step naturally counts cross-inversions. Any problem about counting pairs (i, j) where i < j and f(nums[i], nums[j]) holds can often be solved during merge.

QuickSelect & Partition

Kth Largest / Kth Smallest in O(n) average

// LC 215 Kth Largest Element
// Partition: elements <= pivot on left, > pivot on right
// If pivot lands at index k, done. Otherwise recurse on one side.
int findKthLargest(int[] nums, int k) {
    int target = nums.length - k;  // convert to kth smallest
    return quickSelect(nums, 0, nums.length - 1, target);
}

3-Way Partition (handle duplicates)

// Dutch National Flag: partition into [< pivot | == pivot | > pivot]
int lt = lo, gt = hi, i = lo;
int pivot = nums[lo];
while (i <= gt) {
    if (nums[i] < pivot) swap(nums, lt++, i++);
    else if (nums[i] > pivot) swap(nums, i, gt--);
    else i++;
}

Sweep Line (Event Sorting)

Convert intervals/rectangles into events (start/end), sort events, sweep left-to-right.

Template

// Create events: (time, type) where type: +1=start, -1=end
List<int[]> events = new ArrayList<>();
for (int[] interval : intervals) {
    events.add(new int[]{interval[0], 1});   // open
    events.add(new int[]{interval[1], -1});  // close
}
events.sort((a, b) -> a[0] != b[0] ? a[0] - b[0] : a[1] - b[1]);

int active = 0, maxActive = 0;
for (int[] event : events) {
    active += event[1];
    maxActive = Math.max(maxActive, active);
}

Sorting + Greedy

Sort imposes an order that makes greedy choices provably optimal.

Interval Scheduling (sort by end time)

// LC 435 Non-overlapping Intervals / LC 452 Min Arrows
// Greedy: always pick the interval that ends earliest
Arrays.sort(intervals, (a, b) -> a[1] - b[1]);
int end = Integer.MIN_VALUE, count = 0;
for (int[] iv : intervals) {
    if (iv[0] >= end) { end = iv[1]; count++; }
}

Task Assignment (sort tasks + workers)

// LC 881 Boats to Save People: sort, pair heaviest with lightest
Arrays.sort(people);
int lo = 0, hi = people.length - 1, boats = 0;
while (lo <= hi) {
    if (people[lo] + people[hi] <= limit) lo++;
    hi--;
    boats++;
}

Exchange Argument (prove greedy is optimal)

Technique: Assume two adjacent elements are in wrong order. Show swapping them improves (or doesn't worsen) the answer. This proves the sorted order is optimal.
Examples: LC 179 (largest number), LC 1665 (minimum initial energy)
