Stack vs Queue vs Deque

These three data structures differ in their access pattern. Choosing correctly is the first step to solving the problem efficiently.

StructureOrderOperationsJava Implementation
StackLIFOpush, pop, peekArrayDeque (as stack)
QueueFIFOoffer, poll, peekArrayDeque (as queue)
DequeBoth endsofferFirst/Last, pollFirst/LastArrayDeque

Never use java.util.Stack! It extends Vector (synchronized, slow). Always use ArrayDeque as your stack.

Java ArrayDeque as Stack

// ArrayDeque as Stack (preferred)
Deque<Integer> stack = new ArrayDeque<>();
stack.push(1);       // addFirst internally
stack.pop();         // removeFirst internally
stack.peek();        // peekFirst internally
stack.isEmpty();     // check before pop!

Java ArrayDeque as Queue

// ArrayDeque as Queue (preferred over LinkedList)
Deque<Integer> queue = new ArrayDeque<>();
queue.offer(1);      // addLast internally
queue.poll();         // removeFirst internally
queue.peek();         // peekFirst internally

When to Use Each

Signal in ProblemData Structure
Matching / nesting (parens, tags)Stack
Undo / backtrack / DFSStack
Evaluate expression / calculatorStack (numbers + operators)
Level-order traversal / BFSQueue
Sliding window min/maxMonotone Deque
Next greater/smaller elementMonotone Stack
"Design" with min/max in O(1)Two stacks or augmented stack

Key insight: Stack problems often have O(n) amortized time because each element is pushed and popped at most once.

Parentheses Problems

The stack tracks either characters or indices. Index-based stacks enable computing lengths and positions of valid segments.

Template: Index-based Stack for Parentheses

// Template: stack stores INDICES (not chars)
// Used for: longest valid, min removals, etc.
Deque<Integer> stack = new ArrayDeque<>();
stack.push(-1); // sentinel for length calculation
for (int i = 0; i < s.length(); i++) {
    if (s.charAt(i) == '(') {
        stack.push(i);
    } else {
        stack.pop();
        if (stack.isEmpty()) {
            stack.push(i); // new sentinel
        } else {
            maxLen = Math.max(maxLen, i - stack.peek());
        }
    }
}

Template: Counter-based (no stack needed)

// For simple valid parentheses check
int open = 0;
for (char c : s.toCharArray()) {
    if (c == '(') open++;
    else open--;
    if (open < 0) return false; // too many ')'
}
return open == 0;

LC 20 — Valid Parentheses

public boolean isValid(String s) {
    Deque<Character> stack = new ArrayDeque<>();
    for (char c : s.toCharArray()) {
        if (c == '(') stack.push(')');
        else if (c == '[') stack.push(']');
        else if (c == '{') stack.push('}');
        else if (stack.isEmpty() || stack.pop() != c)
            return false;
    }
    return stack.isEmpty();
}

Trick: Push the expected closing bracket instead of the opening one. Simplifies the matching logic.

LC 1249 — Minimum Remove to Make Valid

public String minRemoveToMakeValid(String s) {
    Deque<Integer> stack = new ArrayDeque<>();
    Set<Integer> remove = new HashSet<>();
    for (int i = 0; i < s.length(); i++) {
        char c = s.charAt(i);
        if (c == '(') stack.push(i);
        else if (c == ')') {
            if (stack.isEmpty()) remove.add(i);
            else stack.pop();
        }
    }
    while (!stack.isEmpty()) remove.add(stack.pop());
    StringBuilder sb = new StringBuilder();
    for (int i = 0; i < s.length(); i++)
        if (!remove.contains(i)) sb.append(s.charAt(i));
    return sb.toString();
}

LC 32 — Longest Valid Parentheses

public int longestValidParentheses(String s) {
    Deque<Integer> stack = new ArrayDeque<>();
    stack.push(-1);
    int max = 0;
    for (int i = 0; i < s.length(); i++) {
        if (s.charAt(i) == '(') {
            stack.push(i);
        } else {
            stack.pop();
            if (stack.isEmpty()) stack.push(i);
            else max = Math.max(max, i - stack.peek());
        }
    }
    return max;
}

LC 22 — Generate Parentheses (Backtracking)

public List<String> generateParenthesis(int n) {
    List<String> res = new ArrayList<>();
    backtrack(res, new StringBuilder(), 0, 0, n);
    return res;
}
private void backtrack(List<String> res, StringBuilder sb, int open, int close, int n) {
    if (sb.length() == 2 * n) { res.add(sb.toString()); return; }
    if (open < n) { sb.append('('); backtrack(res, sb, open+1, close, n); sb.deleteCharAt(sb.length()-1); }
    if (close < open) { sb.append(')'); backtrack(res, sb, open, close+1, n); sb.deleteCharAt(sb.length()-1); }
}

Calculator Problems

Expression evaluation is a classic stack problem. The key patterns are: number stack + sign handling, or recursive descent for nested parentheses.

Template: Basic Calculator (with +, -, parens)

// LC 224 - Basic Calculator I: +, -, (, )
public int calculate(String s) {
    Deque<Integer> stack = new ArrayDeque<>();
    int result = 0, num = 0, sign = 1;
    stack.push(1); // initial sign context
    for (char c : s.toCharArray()) {
        if (Character.isDigit(c)) {
            num = num * 10 + (c - '0');
        } else if (c == '+' || c == '-') {
            result += sign * num;
            num = 0;
            sign = stack.peek() * (c == '+' ? 1 : -1);
        } else if (c == '(') {
            stack.push(sign);
        } else if (c == ')') {
            stack.pop();
        }
    }
    return result + sign * num;
}

Template: Calculator II (with *, / but no parens)

// LC 227 - Basic Calculator II: +, -, *, /
public int calculate(String s) {
    Deque<Integer> stack = new ArrayDeque<>();
    int num = 0;
    char prevOp = '+';
    for (int i = 0; i < s.length(); i++) {
        char c = s.charAt(i);
        if (Character.isDigit(c)) {
            num = num * 10 + (c - '0');
        }
        if ((!Character.isDigit(c) && c != ' ') || i == s.length() - 1) {
            if (prevOp == '+') stack.push(num);
            else if (prevOp == '-') stack.push(-num);
            else if (prevOp == '*') stack.push(stack.pop() * num);
            else if (prevOp == '/') stack.push(stack.pop() / num);
            prevOp = c;
            num = 0;
        }
    }
    int result = 0;
    while (!stack.isEmpty()) result += stack.pop();
    return result;
}

Template: Calculator III (Recursive Descent)

// LC 772 - Basic Calculator III: +, -, *, /, (, )
// Use recursive descent: each '(' triggers a recursive call
int i = 0;
public int calculate(String s) {
    Deque<Integer> stack = new ArrayDeque<>();
    int num = 0;
    char op = '+';
    while (i < s.length()) {
        char c = s.charAt(i++);
        if (Character.isDigit(c)) num = num * 10 + (c - '0');
        if (c == '(') num = calculate(s); // recurse
        if (c == ')' || i == s.length() || (!Character.isDigit(c) && c != ' ')) {
            if (op == '+') stack.push(num);
            else if (op == '-') stack.push(-num);
            else if (op == '*') stack.push(stack.pop() * num);
            else if (op == '/') stack.push(stack.pop() / num);
            op = c; num = 0;
            if (c == ')') break;
        }
    }
    int result = 0;
    for (int x : stack) result += x;
    return result;
}

LC 150 — Evaluate Reverse Polish Notation

public int evalRPN(String[] tokens) {
    Deque<Integer> stack = new ArrayDeque<>();
    for (String t : tokens) {
        if ("+-*/".contains(t)) {
            int b = stack.pop(), a = stack.pop();
            switch (t) {
                case "+": stack.push(a + b); break;
                case "-": stack.push(a - b); break;
                case "*": stack.push(a * b); break;
                case "/": stack.push(a / b); break;
            }
        } else {
            stack.push(Integer.parseInt(t));
        }
    }
    return stack.peek();
}

Pattern: Calculator I = sign stack. Calculator II = prevOp with lazy evaluation. Calculator III = recursive descent or combine both.

Design Problems

These problems test your ability to compose data structures. The trick is maintaining auxiliary state alongside the primary structure.

LC 155 — Min Stack

class MinStack {
    Deque<int[]> stack = new ArrayDeque<>();
    // Each entry: [value, currentMin]

    public void push(int val) {
        int min = stack.isEmpty() ? val : Math.min(val, stack.peek()[1]);
        stack.push(new int[]{val, min});
    }
    public void pop() { stack.pop(); }
    public int top() { return stack.peek()[0]; }
    public int getMin() { return stack.peek()[1]; }
}

LC 716 — Max Stack (with popMax)

// Approach: TreeMap + DoublyLinkedList for O(log n) popMax
class MaxStack {
    TreeMap<Integer, List<Node>> map = new TreeMap<>();
    DoublyLinkedList dll = new DoublyLinkedList();

    public void push(int x) {
        Node node = dll.addLast(x);
        map.computeIfAbsent(x, k -> new ArrayList<>()).add(node);
    }
    public int pop() {
        Node node = dll.removeLast();
        List<Node> list = map.get(node.val);
        list.remove(list.size() - 1);
        if (list.isEmpty()) map.remove(node.val);
        return node.val;
    }
    public int peekMax() { return map.lastKey(); }
    public int popMax() {
        int max = map.lastKey();
        List<Node> list = map.get(max);
        Node node = list.remove(list.size() - 1);
        if (list.isEmpty()) map.remove(max);
        dll.remove(node);
        return max;
    }
}

LC 225 — Stack using Queues

// Push-expensive approach: O(n) push, O(1) pop
class MyStack {
    Queue<Integer> q = new ArrayDeque<>();

    public void push(int x) {
        q.offer(x);
        // Rotate all previous elements to back
        for (int i = 0; i < q.size() - 1; i++)
            q.offer(q.poll());
    }
    public int pop() { return q.poll(); }
    public int top() { return q.peek(); }
    public boolean empty() { return q.isEmpty(); }
}

LC 232 — Queue using Stacks

// Amortized O(1) per operation
class MyQueue {
    Deque<Integer> in = new ArrayDeque<>();
    Deque<Integer> out = new ArrayDeque<>();

    public void push(int x) { in.push(x); }
    public int pop() { transfer(); return out.pop(); }
    public int peek() { transfer(); return out.peek(); }
    public boolean empty() { return in.isEmpty() && out.isEmpty(); }
    private void transfer() {
        if (out.isEmpty())
            while (!in.isEmpty()) out.push(in.pop());
    }
}

Amortized analysis: Each element moves from in to out exactly once, giving O(1) amortized per operation.

LC 895 — Maximum Frequency Stack

class FreqStack {
    Map<Integer, Integer> freq = new HashMap<>();
    Map<Integer, Deque<Integer>> group = new HashMap<>();
    int maxFreq = 0;

    public void push(int val) {
        int f = freq.merge(val, 1, Integer::sum);
        maxFreq = Math.max(maxFreq, f);
        group.computeIfAbsent(f, k -> new ArrayDeque<>()).push(val);
    }
    public int pop() {
        Deque<Integer> stk = group.get(maxFreq);
        int val = stk.pop();
        freq.merge(val, -1, Integer::sum);
        if (stk.isEmpty()) { group.remove(maxFreq); maxFreq--; }
        return val;
    }
}

Monotone Stack

A monotone stack maintains elements in sorted order (increasing or decreasing). When a new element violates the invariant, we pop until the invariant is restored. Each element is pushed/popped at most once: O(n) total.

Template: Monotone Decreasing Stack (Next Greater)

// Find next greater element for each position
// Stack stores INDICES, maintains decreasing order of values
int[] result = new int[n];
Arrays.fill(result, -1);
Deque<Integer> stack = new ArrayDeque<>();
for (int i = 0; i < n; i++) {
    while (!stack.isEmpty() && nums[stack.peek()] < nums[i]) {
        result[stack.pop()] = nums[i];
    }
    stack.push(i);
}

Template: Monotone Increasing Stack (Next Smaller)

// Find next smaller element for each position
// Stack stores INDICES, maintains increasing order of values
int[] result = new int[n];
Arrays.fill(result, -1);
Deque<Integer> stack = new ArrayDeque<>();
for (int i = 0; i < n; i++) {
    while (!stack.isEmpty() && nums[stack.peek()] > nums[i]) {
        result[stack.pop()] = nums[i];
    }
    stack.push(i);
}

Naming confusion: "Monotone decreasing stack" means the stack values decrease from bottom to top. We pop when a LARGER element arrives (to find "next greater"). "Monotone increasing stack" is the reverse.

LC 739 — Daily Temperatures

public int[] dailyTemperatures(int[] temps) {
    int n = temps.length;
    int[] ans = new int[n];
    Deque<Integer> stack = new ArrayDeque<>();
    for (int i = 0; i < n; i++) {
        while (!stack.isEmpty() && temps[stack.peek()] < temps[i]) {
            int j = stack.pop();
            ans[j] = i - j;
        }
        stack.push(i);
    }
    return ans;
}

LC 84 — Largest Rectangle in Histogram

public int largestRectangleArea(int[] heights) {
    int n = heights.length, max = 0;
    Deque<Integer> stack = new ArrayDeque<>();
    for (int i = 0; i <= n; i++) {
        int h = (i == n) ? 0 : heights[i];
        while (!stack.isEmpty() && heights[stack.peek()] > h) {
            int height = heights[stack.pop()];
            int width = stack.isEmpty() ? i : i - stack.peek() - 1;
            max = Math.max(max, height * width);
        }
        stack.push(i);
    }
    return max;
}

Key trick for LC 84: Iterate to i <= n with a virtual bar of height 0 at the end. This forces all remaining bars off the stack.

LC 85 — Maximal Rectangle (extends LC 84)

public int maximalRectangle(char[][] matrix) {
    int m = matrix.length, n = matrix[0].length, max = 0;
    int[] heights = new int[n];
    for (int i = 0; i < m; i++) {
        for (int j = 0; j < n; j++)
            heights[j] = matrix[i][j] == '1' ? heights[j] + 1 : 0;
        max = Math.max(max, largestRectangleArea(heights));
    }
    return max;
}

LC 907 — Sum of Subarray Minimums

public int sumSubarrayMins(int[] arr) {
    int MOD = (int)1e9 + 7, n = arr.length;
    long ans = 0;
    Deque<Integer> stack = new ArrayDeque<>();
    for (int i = 0; i <= n; i++) {
        int val = (i == n) ? 0 : arr[i];
        while (!stack.isEmpty() && arr[stack.peek()] >= val) {
            int mid = stack.pop();
            int left = stack.isEmpty() ? -1 : stack.peek();
            int right = i;
            // arr[mid] is min for (mid-left) * (right-mid) subarrays
            ans = (ans + (long)arr[mid] * (mid - left) * (right - mid)) % MOD;
        }
        stack.push(i);
    }
    return (int)ans;
}

LC 402 — Remove K Digits

public String removeKdigits(String num, int k) {
    Deque<Character> stack = new ArrayDeque<>();
    for (char c : num.toCharArray()) {
        while (k > 0 && !stack.isEmpty() && stack.peek() > c) {
            stack.pop(); k--;
        }
        stack.push(c);
    }
    while (k-- > 0) stack.pop();
    // Build result, skip leading zeros
    StringBuilder sb = new StringBuilder();
    while (!stack.isEmpty()) sb.append(stack.pollLast());
    while (sb.length() > 1 && sb.charAt(0) == '0') sb.deleteCharAt(0);
    return sb.length() == 0 ? "0" : sb.toString();
}

BFS with Queue

BFS uses a queue to process nodes level by level. The key template is the "level-by-level" loop that processes all nodes at the current depth before moving deeper.

Template: Level-by-Level BFS

// Template: process tree/graph level by level
Queue<TreeNode> queue = new ArrayDeque<>();
queue.offer(root);
int depth = 0;
while (!queue.isEmpty()) {
    int size = queue.size(); // MUST capture size before loop
    for (int i = 0; i < size; i++) {
        TreeNode node = queue.poll();
        // process node at current level
        if (node.left != null) queue.offer(node.left);
        if (node.right != null) queue.offer(node.right);
    }
    depth++;
}

Common bug: Forgetting to capture queue.size() before the inner loop. The queue size changes as you add children!

LC 102 — Binary Tree Level Order Traversal

public List<List<Integer>> levelOrder(TreeNode root) {
    List<List<Integer>> res = new ArrayList<>();
    if (root == null) return res;
    Queue<TreeNode> q = new ArrayDeque<>();
    q.offer(root);
    while (!q.isEmpty()) {
        int size = q.size();
        List<Integer> level = new ArrayList<>();
        for (int i = 0; i < size; i++) {
            TreeNode node = q.poll();
            level.add(node.val);
            if (node.left != null) q.offer(node.left);
            if (node.right != null) q.offer(node.right);
        }
        res.add(level);
    }
    return res;
}

LC 103 — Zigzag Level Order Traversal

public List<List<Integer>> zigzagLevelOrder(TreeNode root) {
    List<List<Integer>> res = new ArrayList<>();
    if (root == null) return res;
    Queue<TreeNode> q = new ArrayDeque<>();
    q.offer(root);
    boolean leftToRight = true;
    while (!q.isEmpty()) {
        int size = q.size();
        LinkedList<Integer> level = new LinkedList<>();
        for (int i = 0; i < size; i++) {
            TreeNode node = q.poll();
            if (leftToRight) level.addLast(node.val);
            else level.addFirst(node.val);
            if (node.left != null) q.offer(node.left);
            if (node.right != null) q.offer(node.right);
        }
        res.add(level);
        leftToRight = !leftToRight;
    }
    return res;
}

LC 199 — Binary Tree Right Side View

public List<Integer> rightSideView(TreeNode root) {
    List<Integer> res = new ArrayList<>();
    if (root == null) return res;
    Queue<TreeNode> q = new ArrayDeque<>();
    q.offer(root);
    while (!q.isEmpty()) {
        int size = q.size();
        for (int i = 0; i < size; i++) {
            TreeNode node = q.poll();
            if (i == size - 1) res.add(node.val); // rightmost
            if (node.left != null) q.offer(node.left);
            if (node.right != null) q.offer(node.right);
        }
    }
    return res;
}

LC 994 — Rotting Oranges (Multi-source BFS)

public int orangesRotting(int[][] grid) {
    int m = grid.length, n = grid[0].length;
    Queue<int[]> q = new ArrayDeque<>();
    int fresh = 0;
    for (int i = 0; i < m; i++)
        for (int j = 0; j < n; j++) {
            if (grid[i][j] == 2) q.offer(new int[]{i, j});
            else if (grid[i][j] == 1) fresh++;
        }
    int[][] dirs = {{1,0},{-1,0},{0,1},{0,-1}};
    int minutes = 0;
    while (!q.isEmpty() && fresh > 0) {
        int size = q.size();
        for (int i = 0; i < size; i++) {
            int[] cell = q.poll();
            for (int[] d : dirs) {
                int r = cell[0] + d[0], c = cell[1] + d[1];
                if (r >= 0 && r < m && c >= 0 && c < n && grid[r][c] == 1) {
                    grid[r][c] = 2;
                    fresh--;
                    q.offer(new int[]{r, c});
                }
            }
        }
        minutes++;
    }
    return fresh == 0 ? minutes : -1;
}

Multi-source BFS: Add ALL starting points to the queue first, then BFS. This naturally computes shortest distance from any source. Classic for grid problems.

Deque Applications

Monotone deque maintains a window of candidates in sorted order. As the window slides, we remove elements that fall out of range from the front, and maintain monotonicity from the back.

Template: Monotone Deque (Sliding Window Max)

// Maintain decreasing deque: front is always the max in window
Deque<Integer> dq = new ArrayDeque<>(); // stores indices
int[] result = new int[n - k + 1];
for (int i = 0; i < n; i++) {
    // Remove elements outside window
    while (!dq.isEmpty() && dq.peekFirst() <= i - k)
        dq.pollFirst();
    // Maintain decreasing order (remove smaller from back)
    while (!dq.isEmpty() && nums[dq.peekLast()] <= nums[i])
        dq.pollLast();
    dq.offerLast(i);
    // Window is full
    if (i >= k - 1)
        result[i - k + 1] = nums[dq.peekFirst()];
}

LC 239 — Sliding Window Maximum

public int[] maxSlidingWindow(int[] nums, int k) {
    int n = nums.length;
    int[] res = new int[n - k + 1];
    Deque<Integer> dq = new ArrayDeque<>();
    for (int i = 0; i < n; i++) {
        while (!dq.isEmpty() && dq.peekFirst() <= i - k)
            dq.pollFirst();
        while (!dq.isEmpty() && nums[dq.peekLast()] <= nums[i])
            dq.pollLast();
        dq.offerLast(i);
        if (i >= k - 1) res[i - k + 1] = nums[dq.peekFirst()];
    }
    return res;
}

LC 862 — Shortest Subarray with Sum at Least K

// Key insight: prefix sum + monotone deque
// Need shortest subarray with sum >= k (can have negative numbers)
public int shortestSubarray(int[] nums, int k) {
    int n = nums.length, ans = n + 1;
    long[] prefix = new long[n + 1];
    for (int i = 0; i < n; i++)
        prefix[i + 1] = prefix[i] + nums[i];
    Deque<Integer> dq = new ArrayDeque<>();
    for (int i = 0; i <= n; i++) {
        // If prefix[i] - prefix[dq.front] >= k, update answer
        while (!dq.isEmpty() && prefix[i] - prefix[dq.peekFirst()] >= k)
            ans = Math.min(ans, i - dq.pollFirst());
        // Maintain increasing deque of prefix sums
        while (!dq.isEmpty() && prefix[dq.peekLast()] >= prefix[i])
            dq.pollLast();
        dq.offerLast(i);
    }
    return ans <= n ? ans : -1;
}

Why monotone deque for LC 862? Unlike standard sliding window, negative numbers make the prefix sum non-monotone. The deque maintains an increasing sequence of prefix sums so we can binary-search-style find valid left boundaries.

LC 1425 — Constrained Subsequence Sum

// dp[i] = max sum of subsequence ending at i, with gap <= k
// dp[i] = nums[i] + max(0, max(dp[i-k..i-1]))
// Use monotone deque to get max in sliding window of dp values
public int constrainedSubsetSum(int[] nums, int k) {
    int n = nums.length, ans = Integer.MIN_VALUE;
    int[] dp = new int[n];
    Deque<Integer> dq = new ArrayDeque<>();
    for (int i = 0; i < n; i++) {
        dp[i] = nums[i];
        if (!dq.isEmpty()) dp[i] += Math.max(0, dp[dq.peekFirst()]);
        // Remove out-of-window elements
        while (!dq.isEmpty() && dq.peekFirst() <= i - k)
            dq.pollFirst();
        // Maintain decreasing deque
        while (!dq.isEmpty() && dp[dq.peekLast()] <= dp[i])
            dq.pollLast();
        dq.offerLast(i);
        ans = Math.max(ans, dp[i]);
    }
    return ans;
}

Advanced Stack Problems

These problems use stacks in creative ways: simulating physics, parsing nested structures, maintaining state across events.

LC 735 — Asteroid Collision

public int[] asteroidCollision(int[] asteroids) {
    Deque<Integer> stack = new ArrayDeque<>();
    for (int a : asteroids) {
        boolean alive = true;
        // Collision: stack top is positive, current is negative
        while (alive && !stack.isEmpty() && stack.peek() > 0 && a < 0) {
            if (stack.peek() < -a) stack.pop(); // top explodes
            else if (stack.peek() == -a) { stack.pop(); alive = false; } // both
            else alive = false; // current explodes
        }
        if (alive) stack.push(a);
    }
    int[] res = new int[stack.size()];
    for (int i = res.length - 1; i >= 0; i--) res[i] = stack.pop();
    return res;
}

LC 394 — Decode String

// Iterative approach with two stacks
public String decodeString(String s) {
    Deque<Integer> counts = new ArrayDeque<>();
    Deque<StringBuilder> strings = new ArrayDeque<>();
    StringBuilder cur = new StringBuilder();
    int k = 0;
    for (char c : s.toCharArray()) {
        if (Character.isDigit(c)) {
            k = k * 10 + (c - '0');
        } else if (c == '[') {
            counts.push(k);
            strings.push(cur);
            cur = new StringBuilder();
            k = 0;
        } else if (c == ']') {
            StringBuilder tmp = cur;
            cur = strings.pop();
            int repeat = counts.pop();
            for (int i = 0; i < repeat; i++) cur.append(tmp);
        } else {
            cur.append(c);
        }
    }
    return cur.toString();
}

LC 394 — Decode String (Recursive)

int idx = 0;
public String decodeString(String s) {
    StringBuilder sb = new StringBuilder();
    while (idx < s.length() && s.charAt(idx) != ']') {
        if (Character.isDigit(s.charAt(idx))) {
            int k = 0;
            while (Character.isDigit(s.charAt(idx)))
                k = k * 10 + (s.charAt(idx++) - '0');
            idx++; // skip '['
            String inner = decodeString(s);
            idx++; // skip ']'
            while (k-- > 0) sb.append(inner);
        } else {
            sb.append(s.charAt(idx++));
        }
    }
    return sb.toString();
}

LC 636 — Exclusive Time of Functions

public int[] exclusiveTime(int n, List<String> logs) {
    int[] res = new int[n];
    Deque<int[]> stack = new ArrayDeque<>(); // [id, startTime]
    for (String log : logs) {
        String[] parts = log.split(":");
        int id = Integer.parseInt(parts[0]);
        int time = Integer.parseInt(parts[2]);
        if (parts[1].equals("start")) {
            stack.push(new int[]{id, time});
        } else {
            int[] top = stack.pop();
            int duration = time - top[1] + 1;
            res[id] += duration;
            // Subtract from parent (it was suspended)
            if (!stack.isEmpty()) res[stack.peek()[0]] -= duration;
        }
    }
    return res;
}

LC 901 — Online Stock Span

class StockSpanner {
    Deque<int[]> stack = new ArrayDeque<>(); // [price, span]

    public int next(int price) {
        int span = 1;
        while (!stack.isEmpty() && stack.peek()[0] <= price)
            span += stack.pop()[1];
        stack.push(new int[]{price, span});
        return span;
    }
}

LC 341 — Flatten Nested List Iterator

class NestedIterator implements Iterator<Integer> {
    Deque<NestedInteger> stack = new ArrayDeque<>();

    public NestedIterator(List<NestedInteger> list) {
        // Push in reverse so first element is on top
        for (int i = list.size() - 1; i >= 0; i--)
            stack.push(list.get(i));
    }
    public Integer next() { return stack.pop().getInteger(); }
    public boolean hasNext() {
        while (!stack.isEmpty() && !stack.peek().isInteger()) {
            List<NestedInteger> list = stack.pop().getList();
            for (int i = list.size() - 1; i >= 0; i--)
                stack.push(list.get(i));
        }
        return !stack.isEmpty();
    }
}

Interview Tips

Stack vs Recursion Equivalence

Every recursive solution can be converted to an iterative one using an explicit stack. Google specifically asks candidates to do this conversion.

Google favorite: "Now implement it iteratively." Be ready to convert any DFS/backtracking solution to use an explicit stack.

Converting Recursion to Stack

// Recursive DFS
void dfs(TreeNode node) {
    if (node == null) return;
    process(node);
    dfs(node.left);
    dfs(node.right);
}

// Iterative equivalent (preorder)
void dfsIterative(TreeNode root) {
    Deque<TreeNode> stack = new ArrayDeque<>();
    stack.push(root);
    while (!stack.isEmpty()) {
        TreeNode node = stack.pop();
        if (node == null) continue;
        process(node);
        stack.push(node.right); // right first (LIFO)
        stack.push(node.left);
    }
}

Iterative Inorder (Harder)

// Iterative inorder traversal
Deque<TreeNode> stack = new ArrayDeque<>();
TreeNode cur = root;
while (cur != null || !stack.isEmpty()) {
    while (cur != null) {
        stack.push(cur);
        cur = cur.left;
    }
    cur = stack.pop();
    process(cur); // visit node
    cur = cur.right;
}

When Stack Simulates Recursion

Recursive PatternStack Simulation
Simple DFS (pre-order)Push children in reverse order
In-order traversalPush-left loop + pop + go right
Post-order traversalTwo stacks, or flag-based (prev pointer)
BacktrackingStack of states (path + decision point)
Nested structure (decode string)Stack of partial results

Common Interview Patterns

Pattern 1: Stack for Matching

Parentheses, HTML tags, brackets. Push openers, pop on closers.

Pattern 2: Stack for Monotone Property

Next greater/smaller, histogram, stock prices. Maintain sorted invariant.

Pattern 3: Stack for Undo/State

Calculator, decode string, browser history. Push state before nested call.

Pattern 4: Queue for Level-by-Level

BFS, shortest path in unweighted graph, rotting oranges.

Pattern 5: Deque for Sliding Window Optimization

Replace O(nk) brute force with O(n) monotone deque.
