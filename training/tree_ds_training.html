Binary Search Tree (BST)

In-order traversal of BST gives sorted order. Key property: left < root < right.

BST Validation

boolean isValidBST(TreeNode node, long min, long max) {
    if (node == null) return true;
    if (node.val <= min || node.val >= max) return false;
    return isValidBST(node.left, min, node.val)
        && isValidBST(node.right, node.val, max);
}

Inorder Successor / Predecessor

// Successor: smallest node greater than p
TreeNode successor(TreeNode root, TreeNode p) {
    TreeNode succ = null;
    while (root != null) {
        if (root.val > p.val) { succ = root; root = root.left; }
        else root = root.right;
    }
    return succ;
}

Lowest Common Ancestor in BST

// LC 235: exploit BST property
TreeNode lcaBST(TreeNode root, TreeNode p, TreeNode q) {
    while (root != null) {
        if (p.val < root.val && q.val < root.val) root = root.left;
        else if (p.val > root.val && q.val > root.val) root = root.right;
        else return root;
    }
    return null;
}

// LC 220 Contains Duplicate III: |nums[i]-nums[j]| <= t, |i-j| <= k
TreeSet<Long> set = new TreeSet<>();
for (int i = 0; i < n; i++) {
    Long ceiling = set.ceiling((long)nums[i] - t);
    if (ceiling != null && ceiling <= (long)nums[i] + t)
        return true;
    set.add((long)nums[i]);
    if (set.size() > k) set.remove((long)nums[i - k]);

Pattern: Interval Management with TreeMap

// LC 729 My Calendar: no double-booking
TreeMap<Integer, Integer> calendar = new TreeMap<>();

boolean book(int start, int end) {
    Integer prev = calendar.floorKey(start);
    Integer next = calendar.ceilingKey(start);
    if (prev != null && calendar.get(prev) > start) return false;
    if (next != null && next < end) return false;
    calendar.put(start, end);
    return true;
}

Pattern: Sliding Median / Sliding Window with Sorted Container

// LC 480 Sliding Window Median: two TreeMaps (or two heaps)
// Left half (max): stores smaller half, Right half (min): stores larger half
TreeMap<Integer, Integer> lo = new TreeMap<>(Collections.reverseOrder());
TreeMap<Integer, Integer> hi = new TreeMap<>();
int loSize = 0, hiSize = 0;

void add(int num) {
    lo.merge(num, 1, Integer::sum); loSize++;
    // Move max of lo to hi
    int move = lo.firstKey();
    hi.merge(move, 1, Integer::sum); hiSize++;
    remove(lo, move); loSize--;
    // Balance: lo.size >= hi.size
    if (loSize < hiSize) {
        move = hi.firstKey();
        lo.merge(move, 1, Integer::sum); loSize++;
        remove(hi, move); hiSize--;
    }
}

double getMedian() {
    return loSize > hiSize ? lo.firstKey() : ((long)lo.firstKey() + hi.firstKey()) / 2.0;
}

void remove(TreeMap<Integer,Integer> map, int val) {
    if (map.get(val) == 1) map.remove(val);
    else map.merge(val, -1, Integer::sum);
}

Two-Heap (Running Median)

// LC 295 Find Median from Data Stream
PriorityQueue<Integer> lo = new PriorityQueue<>(Collections.reverseOrder()); // max-heap
PriorityQueue<Integer> hi = new PriorityQueue<>(); // min-heap

void addNum(int num) {
    lo.offer(num);
    hi.offer(lo.poll());
    if (hi.size() > lo.size()) lo.offer(hi.poll());
}

double findMedian() {
    return lo.size() > hi.size() ? lo.peek() : (lo.peek() + hi.peek()) / 2.0;
}

Trie (Prefix Tree)

Implementation

class Trie {
    Trie[] children = new Trie[26];
    boolean isEnd;

    void insert(String word) {
        Trie node = this;
        for (char c : word.toCharArray()) {
            int idx = c - 'a';
            if (node.children[idx] == null)
                node.children[idx] = new Trie();
            node = node.children[idx];
        }
        node.isEnd = true;
    }

    boolean search(String word) {
        Trie node = searchPrefix(word);
        return node != null && node.isEnd;
    }

    boolean startsWith(String prefix) {
        return searchPrefix(prefix) != null;
    }

    Trie searchPrefix(String s) {
        Trie node = this;
        for (char c : s.toCharArray()) {
            node = node.children[c - 'a'];
            if (node == null) return null;
        }
        return node;
    }
}

Trie + DP Pattern (Word Break family)

// LC 139 Word Break: dp[i] = can s[0..i-1] be segmented?
// At each i, walk Trie from i to find all words starting at i
boolean wordBreak(String s, List<String> wordDict) {
    Trie root = buildTrie(wordDict);
    int n = s.length();
    boolean[] dp = new boolean[n + 1];
    dp[0] = true;

    for (int i = 0; i < n; i++) {
        if (!dp[i]) continue;
        Trie node = root;
        for (int j = i; j < n; j++) {
            node = node.children[s.charAt(j) - 'a'];
            if (node == null) break;
            if (node.isEnd) dp[j + 1] = true;
        }
    }
    return dp[n];
}

Segment Tree

Supports range queries (sum, min, max) and point/range updates in O(log n). Build in O(n).

Implementation (Array-based)

class SegTree {
    int[] tree;
    int n;

    SegTree(int[] nums) {
        n = nums.length;
        tree = new int[4 * n];
        build(nums, 1, 0, n - 1);
    }

    void build(int[] nums, int node, int lo, int hi) {
        if (lo == hi) { tree[node] = nums[lo]; return; }
        int mid = (lo + hi) / 2;
        build(nums, 2*node, lo, mid);
        build(nums, 2*node+1, mid+1, hi);
        tree[node] = tree[2*node] + tree[2*node+1];
    }

    void update(int node, int lo, int hi, int idx, int val) {
        if (lo == hi) { tree[node] = val; return; }
        int mid = (lo + hi) / 2;
        if (idx <= mid) update(2*node, lo, mid, idx, val);
        else update(2*node+1, mid+1, hi, idx, val);
        tree[node] = tree[2*node] + tree[2*node+1];
    }

    int query(int node, int lo, int hi, int l, int r) {
        if (r < lo || hi < l) return 0;
        if (l <= lo && hi <= r) return tree[node];
        int mid = (lo + hi) / 2;
        return query(2*node, lo, mid, l, r)
             + query(2*node+1, mid+1, hi, l, r);
    }
}

Range Min/Max Query

// Change merge function from sum to min/max
class SegTreeMin {
    int[] tree;
    int n;

    SegTreeMin(int[] nums) {
        n = nums.length;
        tree = new int[4 * n];
        Arrays.fill(tree, Integer.MAX_VALUE);
        build(nums, 1, 0, n - 1);
    }

    void build(int[] nums, int node, int lo, int hi) {
        if (lo == hi) { tree[node] = nums[lo]; return; }
        int mid = (lo + hi) / 2;
        build(nums, 2*node, lo, mid);
        build(nums, 2*node+1, mid+1, hi);
        tree[node] = Math.min(tree[2*node], tree[2*node+1]);
    }

    void update(int node, int lo, int hi, int idx, int val) {
        if (lo == hi) { tree[node] = val; return; }
        int mid = (lo + hi) / 2;
        if (idx <= mid) update(2*node, lo, mid, idx, val);
        else update(2*node+1, mid+1, hi, idx, val);
        tree[node] = Math.min(tree[2*node], tree[2*node+1]);
    }

    int query(int node, int lo, int hi, int l, int r) {
        if (r < lo || hi < l) return Integer.MAX_VALUE;
        if (l <= lo && hi <= r) return tree[node];
        int mid = (lo + hi) / 2;
        return Math.min(
            query(2*node, lo, mid, l, r),
            query(2*node+1, mid+1, hi, l, r));
    }
}
// For max: replace Math.min → Math.max, MAX_VALUE → MIN_VALUE

Lazy Propagation (Range Update + Range Query)

Problem: "Add val to all elements in [l, r]" + "Query sum of [l, r]"
Key idea: Don't push updates immediately to children. Store pending updates in a lazy[] array. Push down only when needed (during query or child update).

class LazySegTree {
    long[] tree, lazy;
    int n;

    LazySegTree(int[] nums) {
        n = nums.length;
        tree = new long[4 * n];
        lazy = new long[4 * n];
        build(nums, 1, 0, n - 1);
    }

    void build(int[] nums, int node, int lo, int hi) {
        if (lo == hi) { tree[node] = nums[lo]; return; }
        int mid = (lo + hi) / 2;
        build(nums, 2*node, lo, mid);
        build(nums, 2*node+1, mid+1, hi);
        tree[node] = tree[2*node] + tree[2*node+1];
    }

    void pushDown(int node, int lo, int hi) {
        if (lazy[node] != 0) {
            int mid = (lo + hi) / 2;
            apply(2*node, lo, mid, lazy[node]);
            apply(2*node+1, mid+1, hi, lazy[node]);
            lazy[node] = 0;
        }
    }

    void apply(int node, int lo, int hi, long val) {
        tree[node] += val * (hi - lo + 1);  // sum += val * count
        lazy[node] += val;
    }

    // Range update: add val to all elements in [l, r]
    void update(int node, int lo, int hi, int l, int r, long val) {
        if (r < lo || hi < l) return;
        if (l <= lo && hi <= r) { apply(node, lo, hi, val); return; }
        pushDown(node, lo, hi);
        int mid = (lo + hi) / 2;
        update(2*node, lo, mid, l, r, val);
        update(2*node+1, mid+1, hi, l, r, val);
        tree[node] = tree[2*node] + tree[2*node+1];
    }

    // Range query: sum of [l, r]
    long query(int node, int lo, int hi, int l, int r) {
        if (r < lo || hi < l) return 0;
        if (l <= lo && hi <= r) return tree[node];
        pushDown(node, lo, hi);
        int mid = (lo + hi) / 2;
        return query(2*node, lo, mid, l, r)
             + query(2*node+1, mid+1, hi, l, r);
    }
}

Lazy for Range Set (assign val to [l, r])

// Instead of adding, overwrite: tree[node] = val * (hi - lo + 1)
// lazy[node] = val (use a sentinel like Long.MIN_VALUE for "no pending")
void apply(int node, int lo, int hi, long val) {
    tree[node] = val * (hi - lo + 1);
    lazy[node] = val;  // overwrite (not add)
}

Sparse Table (Static RMQ in O(1))

When array is immutable (no updates), Sparse Table gives O(1) range min/max after O(n log n) preprocessing. Faster than Segment Tree for static queries.

// sparse[k][i] = min of range [i, i + 2^k - 1]
int[][] sparse;
int[] log;

void buildSparse(int[] arr) {
    int n = arr.length, K = Integer.numberOfTrailingZeros(Integer.highestOneBit(n)) + 1;
    sparse = new int[K][n];
    sparse[0] = arr.clone();
    for (int k = 1; k < K; k++)
        for (int i = 0; i + (1 << k) <= n; i++)
            sparse[k][i] = Math.min(sparse[k-1][i], sparse[k-1][i + (1 << (k-1))]);
}

// O(1) query: overlap two precomputed ranges
int queryMin(int l, int r) {
    int k = 31 - Integer.numberOfLeadingZeros(r - l + 1);
    return Math.min(sparse[k][l], sparse[k][r - (1 << k) + 1]);
}

Binary Indexed Tree (Fenwick Tree)

Simpler than Segment Tree. Supports prefix sum query and point update in O(log n). Preferred when you only need prefix operations.

Implementation

class BIT {
    int[] tree;
    int n;

    BIT(int n) {
        this.n = n;
        tree = new int[n + 1];
    }

    void update(int i, int delta) {
        for (; i <= n; i += i & (-i))
            tree[i] += delta;
    }

    int query(int i) {
        int sum = 0;
        for (; i > 0; i -= i & (-i))
            sum += tree[i];
        return sum;
    }

    int rangeQuery(int l, int r) {
        return query(r) - query(l - 1);
    }
}

BIT for Max (instead of sum)

void update(int i, int val) {
    for (; i <= n; i += i & (-i))
        tree[i] = Math.max(tree[i], val);
}

int query(int i) {
    int res = 0;
    for (; i > 0; i -= i & (-i))
        res = Math.max(res, tree[i]);
    return res;
}

BIT for (length, count) pair — Number of LIS (LC 673)

Idea: Store (maxLen, count) at each BIT position. Query prefix max-length and sum of counts that achieve it. This solves "Number of Longest Increasing Subsequence" in O(n log n).

// LC 673: Number of Longest Increasing Subsequence
// BIT stores (len, cnt) pairs — merge by taking max-len and summing counts
int[] bitLen, bitCnt;  // parallel arrays

void update(int i, int len, int cnt) {
    for (; i <= n; i += i & (-i)) {
        if (bitLen[i] < len) {
            bitLen[i] = len;
            bitCnt[i] = cnt;
        } else if (bitLen[i] == len) {
            bitCnt[i] += cnt;
        }
    }
}

// Query prefix [1, i]: return (maxLen, totalCount)
int[] query(int i) {
    int len = 0, cnt = 0;
    for (; i > 0; i -= i & (-i)) {
        if (bitLen[i] > len) {
            len = bitLen[i];
            cnt = bitCnt[i];
        } else if (bitLen[i] == len) {
            cnt += bitCnt[i];
        }
    }
    return new int[]{len, cnt};
}

int findNumberOfLIS(int[] nums) {
    int m = nums.length;
    // Coordinate compression via index sorting
    // Sort indices by value; rank = position in sorted order
    Integer[] idx = new Integer[m];
    for (int i = 0; i < m; i++) idx[i] = i;
    Arrays.sort(idx, (a, b) -> nums[a] != nums[b] ? nums[a] - nums[b] : a - b);

    int[] rank = new int[m];
    for (int i = 0; i < m; i++) {
        // Same value gets same rank to handle duplicates in strict LIS
        rank[idx[i]] = (i > 0 && nums[idx[i]] == nums[idx[i-1]])
            ? rank[idx[i-1]] : i + 1;
    }

    n = m;
    bitLen = new int[n + 1];
    bitCnt = new int[n + 1];

    int maxLen = 0, ans = 0;
    for (int i = 0; i < m; i++) {
        int v = rank[i];
        int[] prev = query(v - 1);  // best LIS for values strictly < nums[i]
        int curLen = prev[0] + 1;
        int curCnt = Math.max(prev[1], 1);
        update(v, curLen, curCnt);
        if (curLen > maxLen) { maxLen = curLen; ans = curCnt; }
        else if (curLen == maxLen) ans += curCnt;
    }
    return ans;
}

BIT Binary Search: Find k-th Element in Dynamic Set

Pattern: "Find the k-th remaining/available element" in a set that changes dynamically (elements removed or added). BIT stores 1 for present, 0 for removed. Binary search on prefix sum to find position where prefix == k.

// BIT Binary Search: find smallest pos where prefixSum(pos) == target
// O(log n) — no linear scan needed
int findKth(int[] bit, int n, int target) {
    int pos = 0;
    for (int mask = Integer.highestOneBit(n); mask > 0; mask >>= 1) {
        if (pos + mask <= n && bit[pos + mask] < target) {
            target -= bit[pos + mask];
            pos += mask;
        }
    }
    return pos + 1; // 1-indexed position
}

// Usage: Initialize BIT with all 1s (all slots available)
int[] bit = new int[n + 1];
for (int i = 1; i <= n; i++) {
    bit[i] += 1;
    int j = i + (i & -i);
    if (j <= n) bit[j] += bit[i];
}

// Find k-th available slot, then mark it as taken
int pos = findKth(bit, n, k);
for (int i = pos; i <= n; i += i & -i) bit[i]--; // remove from set

Why this works

The BIT stores prefix count of available elements. Binary search descends bit by bit (like binary search on a sorted array) — at each level, if the left half has fewer than target elements, skip to the right half. This gives O(log n) per query instead of O(n) linear scan.

Binary Lifting + LCA (Lowest Common Ancestor)

Binary Lifting: Precompute up[v][k] = 2^k-th ancestor of v. Enables jumping to any ancestor in O(log n).
LCA: Bring both nodes to same depth, then lift both until they meet. O(log n) per query after O(n log n) preprocessing.

int LOG = 20;  // log2(max_n)
int[][] up;    // up[v][k] = 2^k-th ancestor of v
int[] depth;

// Preprocessing: O(n log n)
void preprocess(int n, List<List<Integer>> adj) {
    up = new int[n][LOG];
    depth = new int[n];
    dfs(0, 0, 0, adj);
    // Fill sparse table
    for (int k = 1; k < LOG; k++)
        for (int v = 0; v < n; v++)
            up[v][k] = up[up[v][k-1]][k-1];
}

void dfs(int v, int parent, int d, List<List<Integer>> adj) {
    up[v][0] = parent;
    depth[v] = d;
    for (int u : adj.get(v))
        if (u != parent) dfs(u, v, d + 1, adj);
}

// LCA query: O(log n)
int lca(int u, int v) {
    // Step 1: bring to same depth
    if (depth[u] < depth[v]) { int t = u; u = v; v = t; }
    int diff = depth[u] - depth[v];
    for (int k = 0; k < LOG; k++)
        if (((diff >> k) & 1) == 1) u = up[u][k];

    if (u == v) return u;

    // Step 2: lift both until just below LCA
    for (int k = LOG - 1; k >= 0; k--)
        if (up[u][k] != up[v][k]) { u = up[u][k]; v = up[v][k]; }

    return up[u][0];
}

Kth Ancestor (LC 1483)

// Jump k steps up from node using binary lifting
int getKthAncestor(int node, int k) {
    for (int i = 0; i < LOG && node != -1; i++)
        if (((k >> i) & 1) == 1) node = up[node][i];
    return node;
}

Euler Tour (tin/tout) + O(1) Ancestor Check

Key insight (cp-algorithms style): During DFS, record entry time tin[v] and exit time tout[v]. Node u is an ancestor of v if and only if tin[u] ≤ tin[v] && tout[v] ≤ tout[u]. This is O(1) after O(n) preprocessing.

int[] tin, tout;
int timer = 0;

void eulerTour(int v, int parent, List<List<Integer>> adj) {
    tin[v] = timer++;
    up[v][0] = parent;
    for (int k = 1; k < LOG; k++)
        up[v][k] = up[up[v][k-1]][k-1];

    for (int u : adj.get(v))
        if (u != parent) eulerTour(u, v, adj);

    tout[v] = timer++;
}

// O(1) ancestor check
boolean isAncestor(int u, int v) {
    return tin[u] <= tin[v] && tout[v] <= tout[u];
}

Full LCA using Euler Tour + Binary Lifting (cp-algorithms template)

int LOG = 20;
int[][] up;
int[] tin, tout, depth;
int timer = 0;

void init(int n, List<List<Integer>> adj) {
    up = new int[n][LOG];
    tin = new int[n];
    tout = new int[n];
    depth = new int[n];
    dfs(0, 0, 0, adj);
}

void dfs(int v, int parent, int d, List<List<Integer>> adj) {
    tin[v] = timer++;
    depth[v] = d;
    up[v][0] = parent;
    for (int k = 1; k < LOG; k++)
        up[v][k] = up[up[v][k-1]][k-1];
    for (int u : adj.get(v))
        if (u != parent) dfs(u, v, d + 1, adj);
    tout[v] = timer++;
}

boolean isAncestor(int u, int v) {
    return tin[u] <= tin[v] && tout[v] <= tout[u];
}

int lca(int u, int v) {
    if (isAncestor(u, v)) return u;
    if (isAncestor(v, u)) return v;
    // Lift u until its ancestor covers v
    for (int k = LOG - 1; k >= 0; k--)
        if (!isAncestor(up[u][k], v))
            u = up[u][k];
    return up[u][0];
}

Why this version is elegant: The LCA function uses isAncestor() directly in binary lifting. Instead of equalizing depths first, it simply lifts u until its ancestor contains v. Cleaner logic, same O(log n).

Distance Between Two Nodes

// dist(u, v) = depth[u] + depth[v] - 2 * depth[lca(u, v)]
int dist(int u, int v) {
    return depth[u] + depth[v] - 2 * depth[lca(u, v)];
}

// Euler Tour maps subtree(v) to contiguous range [tin[v], tout[v]]
// This enables using BIT/SegTree for subtree sum/update:
//   - "Add x to all nodes in subtree(v)" → range update [tin[v], tout[v]]
//   - "Sum of values in subtree(v)" → range query [tin[v], tout[v]]
// Flatten tree into array order[tin[v]] = v, then operate on the array

Applications

Create a new "version" of the tree on each update by only creating new nodes on the path from root to updated leaf. Shares unchanged subtrees. Used for "Kth smallest in range" (LC 2286).

Skip List

A probabilistic data structure that provides O(log n) search, insert, and delete — like a balanced BST but much simpler to implement. Used by Redis for its Sorted Set (ZSET) implementation.

Why Skip List Over BST?

AspectSkip ListBalanced BST (AVL/Red-Black)
Implementation complexitySimple (pointer manipulation)Complex (rotations, color/height tracking)
Expected timeO(log n) all opsO(log n) all ops
Range queriesNatural (linked list at bottom)In-order traversal
ConcurrencyEasy to make lock-freeHard (rotation conflicts)
SpaceO(n) expected (2n pointers avg)O(n) (3 pointers per node)
Cache behaviorGood (sequential at bottom level)Poor (pointer chasing)

Why Redis chose Skip List: (1) Simple to implement correctly. (2) Easy to support ZRANGEBYSCORE (range queries by traversing forward pointers). (3) Lock-free concurrent variant (ConcurrentSkipListMap in Java). (4) Comparable performance to balanced BST in practice.

Structure

Multiple sorted linked lists layered on top of each other. Bottom layer = all elements. Each higher layer is a "fast lane" containing a random subset (~50%) of the layer below.

// Conceptual picture (MAX_LEVEL = 4):
// Level 3:  HEAD ─────────────────────── 50 ──────── NIL
// Level 2:  HEAD ──── 10 ─────────────── 50 ──── 70  NIL
// Level 1:  HEAD ──── 10 ──── 30 ─────── 50 ──── 70  NIL
// Level 0:  HEAD ─ 5 ─ 10 ─ 20 ─ 30 ─ 40 ─ 50 ─ 70 ─ 90  NIL

Java Implementation (LC 1206)

class Skiplist {
    private static final int MAX_LEVEL = 16;
    private static final double P = 0.5;
    private Node head = new Node(-1, MAX_LEVEL);
    private int level = 0;
    private Random rand = new Random();

    private int randomLevel() {
        int lvl = 0;
        while (lvl < MAX_LEVEL - 1 && rand.nextDouble() < P) lvl++;
        return lvl;
    }

    public boolean search(int target) {
        Node curr = head;
        for (int i = level; i >= 0; i--)
            while (curr.next[i] != null && curr.next[i].val < target)
                curr = curr.next[i];
        curr = curr.next[0];
        return curr != null && curr.val == target;
    }

    public void add(int num) {
        Node[] update = new Node[MAX_LEVEL];
        Node curr = head;
        for (int i = level; i >= 0; i--) {
            while (curr.next[i] != null && curr.next[i].val < num)
                curr = curr.next[i];
            update[i] = curr;
        }
        int newLevel = randomLevel();
        if (newLevel > level) {
            for (int i = level + 1; i <= newLevel; i++) update[i] = head;
            level = newLevel;
        }
        Node newNode = new Node(num, newLevel + 1);
        for (int i = 0; i <= newLevel; i++) {
            newNode.next[i] = update[i].next[i];
            update[i].next[i] = newNode;
        }
    }

    public boolean erase(int num) {
        Node[] update = new Node[MAX_LEVEL];
        Node curr = head;
        for (int i = level; i >= 0; i--) {
            while (curr.next[i] != null && curr.next[i].val < num)
                curr = curr.next[i];
            update[i] = curr;
        }
        curr = curr.next[0];
        if (curr == null || curr.val != num) return false;
        for (int i = 0; i <= level; i++) {
            if (update[i].next[i] != curr) break;
            update[i].next[i] = curr.next[i];
        }
        while (level > 0 && head.next[level] == null) level--;
        return true;
    }

    private class Node {
        int val;
        Node[] next;
        Node(int val, int levels) { this.val = val; next = new Node[levels]; }
    }
}

Key Operations Explained

Search: "Highway exit" pattern

Start at top level. Move right while next value < target. Drop down one level. Repeat. Like taking highways then local roads.

Insert: Random promotion

After finding the position (same search logic), flip coins to determine the new node's level. Each level has 50% chance (P=0.5). On average, a node appears in 2 levels.

Delete: Update all level pointers

Track the "update" array (last node at each level before the target), then unlink the target at every level it appears.

Redis ZSET Use Case

// Redis Sorted Set operations mapped to skip list:
// ZADD key score member   → skiplist.add(score, member)
// ZREM key member         → skiplist.erase(member)
// ZRANK key member        → count nodes before member (traverse + count)
// ZRANGEBYSCORE min max   → find min, traverse forward until > max
// ZREVRANGE               → backward pointers (doubly-linked at bottom level)

Redis optimization: Redis's skip list is doubly-linked at level 0 (backward pointer) for reverse traversal, and stores a span field at each forward pointer to enable O(log n) rank queries without counting.
